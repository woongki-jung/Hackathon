# 용어 분석 정책 정의

## 개요
- Claude API를 활용한 메일 내용 분석, 용어 추출, 해설 생성에 관한 비즈니스 규칙을 정의한다.
- PRD의 "백엔드 기능 > 2. 내용 분석" 요구사항에 기반한다.
- 분석 결과의 품질과 일관성을 보장하기 위한 규칙을 명시한다.

---

## POL-TERM 용어 분석 정책

### 기본 정보
| 항목 | 내용 |
|------|------|
| 정책명 | 용어 분석 정책 |
| 분류 | 비즈니스 규칙 |
| 적용 범위 | 메일 내용 분석, 용어 추출/분류, 해설 생성, 용어 해설집 파일 관리 |
| 우선순위 | 보통 |
| 관련 정책 | POL-AUTH (Claude API 키 관리), POL-MAIL (분석 대상 파일 제공), POL-DATA (분석 결과 저장) |

### 정책 규칙

#### 1. 분석 대상 및 트리거

| 규칙 번호 | 규칙 내용 | 적용 조건 | 위반 시 처리 |
|-----------|-----------|-----------|--------------|
| TERM-R-001 | 메일 수신 프로세스에서 MAIL_STORAGE_PATH에 새로 저장된 텍스트 파일을 분석 대상으로 한다. | 메일 수신 후 | - |
| TERM-R-002 | 분석은 메일 수신 주기와 동일한 주기로 실행하며, 메일 수신 완료 후 순차적으로 수행한다. | 메일 수신 완료 후 | - |
| TERM-R-003 | 이미 분석 완료된 파일은 재분석하지 않는다. 분석 완료 여부는 DB에 기록하여 관리한다. | 분석 대상 파일 선정 시 | - |

#### 2. Claude API 호출

| 규칙 번호 | 규칙 내용 | 적용 조건 | 위반 시 처리 |
|-----------|-----------|-----------|--------------|
| TERM-R-004 | Claude API 호출에는 @anthropic-ai/sdk 라이브러리를 사용한다. | API 호출 시 | - |
| TERM-R-005 | 사용 모델은 환경변수 ANTHROPIC_MODEL(기본값: claude-sonnet-4-6)로 설정한다. | API 호출 시 | - |
| TERM-R-006 | API 키는 환경변수 ANTHROPIC_API_KEY에서 읽는다. 미설정 시 분석 프로세스를 건너뛴다. | 분석 프로세스 시작 시 | 건너뜀 로그 기록, 서비스 상태에 "API 키 미설정" 표시 |
| TERM-R-007 | API 호출 실패 시 최대 2회 재시도하며, 재시도 간격은 5초로 한다. | API 호출 실패 시 | 3회 모두 실패 시 해당 파일 분석 건너뜀, 오류 로그 기록 |
| TERM-R-008 | 단일 API 호출의 타임아웃은 60초로 설정한다. | API 호출 시 | 타임아웃 발생 시 재시도 로직 적용 |
| TERM-R-009 | API 호출 시 전송하는 메일 본문은 최대 10,000자로 제한한다. 초과분은 잘라낸다. | API 요청 생성 시 | 잘린 내용에 대한 별도 알림 없음 |

| 규칙 번호 | 규칙 내용 | 적용 조건 | 위반 시 처리 |
|-----------|-----------|-----------|--------------|
| TERM-R-010 | 각 메일에 대해 핵심 내용 요약(최대 500자)을 생성한다. | 메일 분석 시 | - |
| TERM-R-011 | 각 메일에 대해 후속 작업 제안 목록(최대 5개 항목)을 생성한다. | 메일 분석 시 | - |
| TERM-R-012 | 메일 요약과 후속 작업 제안은 DB에 저장하며, 메일 원본 파일과 연결(파일명 참조)한다. | 분석 결과 저장 시 | - |

#### 4. 용어 추출 및 분류

| 규칙 번호 | 규칙 내용 | 적용 조건 | 위반 시 처리 |
|-----------|-----------|-----------|--------------|
| TERM-R-013 | 메일 본문에서 EMR 시스템 관련 용어, 비즈니스 용어, 약어를 추출하여 분류한다. | 메일 분석 시 | - |
| TERM-R-014 | 추출된 각 용어에 대해 한국어 해설을 생성한다. | 용어 추출 후 | - |
| TERM-R-015 | 1건의 메일에서 추출하는 용어 수는 최대 30개로 제한한다. | 용어 추출 시 | 30개 초과 시 관련도 높은 순서로 30개 선택 |
| TERM-R-016 | 추출된 용어와 해설 쌍은 DB에 저장하고, 동시에 용어별 마크다운 파일(POL-DATA DATA-R-011 규칙)로도 저장한다. | 용어 분석 결과 저장 시 | - |

|-----------|-----------|-----------|--------------|
| TERM-R-017 | 용어 해설집 파일은 마크다운 형식으로, 최소한 용어명(제목)과 해설(본문)을 포함한다. | 용어 파일 생성/갱신 시 | - |
| TERM-R-018 | 기존에 동일 용어 파일이 있는 경우, 새로운 해설이 기존 해설보다 더 상세하거나 정확하면 갱신한다. 갱신 판단은 새 해설의 문자 수가 기존 대비 20% 이상 증가했거나, 새 출처 메일 정보가 추가된 경우로 한다. | 중복 용어 파일 발견 시 | - |

| TERM-R-021 | 분석 실패한 파일은 다음 주기에 최대 2회까지 재시도한다. 3회 연속 실패 시 해당 파일의 상태를 "실패"로 확정한다. | 분석 실패 시 | 실패 확정 로그 기록 |

### 예외 사항
- ANTHROPIC_API_KEY가 설정되지 않은 경우, 메일 수신은 정상 수행하되 분석 프로세스만 건너뛴다.
- 메일 본문이 비어있거나 100자 미만인 경우, 용어 분석을 건너뛰고 요약만 저장한다.
- 영문 전용 메일도 분석 대상에 포함하며, 해설은 한국어로 생성한다.

### 구현 가이드
- Claude API SDK: @anthropic-ai/sdk를 사용하여 메시지 API를 호출한다.
- 프롬프트 설계: 용어 추출, 분류, 해설 생성을 위한 시스템 프롬프트를 별도 파일로 관리하여 유지보수성을 확보한다.
- 병렬 처리: 여러 메일 파일의 분석은 순차 처리를 기본으로 하되, API rate limit에 여유가 있는 경우 병렬 처리를 검토할 수 있다.

### 관련 문서
- 관련 데이터 정의: `docs/specs/datas/` (용어 테이블, 메일 분석 결과 테이블)
- 관련 기능 정의: `docs/specs/functions/` (용어 분석 기능, 해설 생성 기능)
- 관련 화면 정의: `docs/specs/screens/` (용어사전 뷰어, 업무지원 페이지)
