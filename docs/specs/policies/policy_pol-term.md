# POL-TERM: 용어 분석 정책

## 정책 개요

| 항목 | 내용 |
|------|------|
| 정책 코드 | POL-TERM |
| 정책명 | 용어 분석 정책 |
| 적용 대상 | 메일 내용 분석, 용어 분류, 해설 생성, 사전 관리 |
| 관련 PRD 기능 | 용어사전 생성 기능 |
| 의존 정책 | POL-DATA, POL-AUTH |

## 정책 상세

### TERM-01: 용어 분류 기준

- 분석 대상 용어를 다음 카테고리로 분류한다:
  - **EMR**: EMR(전자의무기록) 시스템 관련 용어 (예: OCS, PACS, CDR, HL7, FHIR 등)
  - **Business**: 의료/병원 업무 도메인 용어 (예: DRG, 수가, 원외처방 등)
  - **Abbreviation**: 업무에서 사용되는 약어/축약어 (예: OP, PT, EMR 등)
- 일반적인 영어 단어, 관사, 전치사 등은 분석 대상에서 제외한다.
- 2글자 미만의 단독 문자열은 분석 대상에서 제외한다.

### TERM-02: 용어 추출 규칙

- 메일 제목과 본문에서 다음 패턴의 용어를 추출한다:
  - 대문자 약어 (2글자 이상 연속 대문자: `[A-Z]{2,}`)
  - 영문+숫자 조합의 코드성 용어 (예: ICD-10, HL7v2)
  - 한글 전문 용어 (추후 사용자가 등록한 용어 패턴 목록과 매칭)
- 불용어(stop words) 목록을 관리하여 일반적인 영어 약어(IT, OK, PM, AM 등)를 필터링한다.
- 불용어 목록은 사용자가 편집 가능하도록 설정 파일로 관리한다.

### TERM-03: 해설 생성 규칙

- 추출된 용어에 대해 Claude API를 호출하여 해설을 생성한다.
- 해설 생성 프롬프트에는 다음 컨텍스트를 포함한다:
  - 용어가 발견된 메일의 제목과 본문 일부 (해당 용어 전후 200자)
  - 용어의 카테고리 (EMR/Business/Abbreviation)
- 해설 형식:
  - 1줄 요약 (50자 이내)
  - 상세 설명 (200자 이내)
  - 관련 용어 목록 (있는 경우)
- API 호출 비용 절감을 위해 이미 사전에 등록된 용어는 해설을 재생성하지 않는다.

### TERM-04: 중복 용어 처리 정책

- 이미 사전에 존재하는 용어가 재발견된 경우:
  - `source_count`를 1 증가시킨다.
  - `last_seen_at`을 현재 시각으로 갱신한다.
  - `source_files`에 출처 파일을 추가한다 (최대 10건 유지).
  - 해설 내용은 변경하지 않는다.
- 동일 용어가 다른 카테고리로 분류될 가능성이 있는 경우, 기존 카테고리를 유지한다.

### TERM-05: 배치 처리 정책

- 분석 요청 폴더의 파일을 일괄 처리한다.
- 처리 순서: 파일 생성 시간 기준 오름차순 (오래된 것부터).
- 1회 배치 처리 시 최대 10개 파일을 처리한다.
- 파일당 최대 50개 용어를 추출한다 (초과 시 빈도순 상위 50개).
- Claude API 호출은 1회 배치당 최대 20건으로 제한한다 (Rate Limit 및 비용 고려).

### TERM-06: 분석 실패 처리

- 파일 파싱 실패: 에러 로그 기록 후 해당 파일을 `Error/` 디렉터리로 이동
- Claude API 호출 실패: 해당 용어를 "해설 미완료" 상태로 사전에 등록하고, 다음 배치에서 재시도
- 부분 실패: 성공한 용어는 저장하고, 실패한 용어만 재시도 큐에 추가

## 제약 사항

- Claude API 호출 비용을 고려하여 일일 최대 호출 횟수를 200건으로 제한한다 (설정 가능).
- 용어 사전의 최대 항목 수는 제한하지 않으나, 10,000건 초과 시 성능 모니터링을 권장한다.
- 해설 생성 시 환자 개인정보(이름, 주민번호 등)가 포함되지 않도록 사전 필터링한다.
