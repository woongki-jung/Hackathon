# 인증/보안 정책 정의

## 개요
- 웹 서비스 접근 제어, 사용자 인증, 세션 관리, 민감정보 보호에 관한 정책을 정의한다.
- 관리자가 등록한 사용자만 서비스에 접근할 수 있는 폐쇄형 인증 체계를 적용한다.
- 외부 서비스(IMAP, Claude API) 인증 정보의 안전한 저장 및 사용 규칙을 정의한다.

---

## POL-AUTH 인증/보안 정책

### 기본 정보
| 항목 | 내용 |
|------|------|
| 정책명 | 인증/보안 정책 |
| 분류 | 인증/인가, 보안 |
| 적용 범위 | 로그인 페이지, 모든 인증 필요 API 라우트, 환경설정 페이지, .env.local 관리 |
| 우선순위 | 높음 |
| 관련 정책 | POL-DATA (사용자 계정 저장), POL-MAIL (IMAP 인증 정보), POL-TERM (Claude API 키) |

### 정책 규칙

#### 1. 계정 관리

| 규칙 번호 | 규칙 내용 | 적용 조건 | 위반 시 처리 |
|-----------|-----------|-----------|--------------|
| AUTH-R-001 | 시스템 최초 실행 시 환경변수(ADMIN_USERNAME, ADMIN_PASSWORD)로 관리자 계정 1개를 자동 생성한다. | 서버 시작 시 관리자 계정이 DB에 없는 경우 | 서버 시작 로그에 경고 기록, 관리자 계정 없이 서비스 운영 불가 |
| AUTH-R-002 | 관리자만 새 사용자 계정을 등록할 수 있다. | 사용자 등록 API 호출 시 | 403 Forbidden 반환 |
| AUTH-R-003 | 사용자 아이디는 4자 이상 20자 이하의 영문 소문자, 숫자, 언더스코어(_)만 허용한다. | 계정 생성 시 | 400 Bad Request, 유효성 오류 메시지 반환 |
| AUTH-R-004 | 사용자 아이디는 시스템 내에서 고유해야 한다. | 계정 생성 시 | 409 Conflict, 중복 메시지 반환 |

#### 2. 비밀번호 정책

| 규칙 번호 | 규칙 내용 | 적용 조건 | 위반 시 처리 |
|-----------|-----------|-----------|--------------|
| AUTH-R-005 | 비밀번호는 8자 이상이어야 한다. | 계정 생성 및 비밀번호 변경 시 | 400 Bad Request, 유효성 오류 메시지 반환 |
| AUTH-R-006 | 비밀번호는 영문자, 숫자, 특수문자를 각각 1자 이상 포함해야 한다. | 계정 생성 및 비밀번호 변경 시 | 400 Bad Request, 유효성 오류 메시지 반환 |
| AUTH-R-007 | 비밀번호는 bcrypt 알고리즘(salt rounds: 10)으로 해싱하여 저장한다. | 비밀번호 저장 시 | 평문 저장 금지, 구현 검증 필수 |

#### 3. 로그인/세션 관리

| 규칙 번호 | 규칙 내용 | 적용 조건 | 위반 시 처리 |
|-----------|-----------|-----------|--------------|
| AUTH-R-008 | 로그인은 아이디/비밀번호 방식으로만 수행한다. | 로그인 API 호출 시 | - |
| AUTH-R-009 | 세션은 iron-session 기반 암호화된 HTTP-only 쿠키로 관리한다. | 로그인 성공 후 | - |
| AUTH-R-010 | 세션 유효 기간은 24시간으로 설정한다. 만료 시 재로그인이 필요하다. | 모든 인증 필요 요청 시 | 401 Unauthorized 반환, 로그인 페이지로 리다이렉트 |
| AUTH-R-011 | 로그인 실패 시 "아이디 또는 비밀번호가 일치하지 않습니다"라는 통합 메시지를 반환한다. 아이디/비밀번호 중 어느 쪽이 틀렸는지 구분하여 알리지 않는다. | 로그인 실패 시 | 401 Unauthorized, 통합 오류 메시지 반환 |
| AUTH-R-012 | 인증이 필요한 페이지/API에 미인증 상태로 접근 시 로그인 페이지로 리다이렉트한다. | 세션 없는 상태에서 보호된 리소스 접근 시 | 로그인 페이지로 302 리다이렉트 (페이지) 또는 401 반환 (API) |

#### 4. 접근 제어

| 규칙 번호 | 규칙 내용 | 적용 조건 | 위반 시 처리 |
|-----------|-----------|-----------|--------------|
| AUTH-R-013 | 역할은 관리자(admin)와 일반 사용자(user) 2가지로 구분한다. | 모든 인증 필요 요청 시 | - |
| AUTH-R-014 | 환경설정(메일 서버 정보, 분석 설정) 변경은 관리자만 가능하다. | 환경설정 API 호출 시 | 403 Forbidden 반환 |
| AUTH-R-015 | 사용자 계정 관리(등록, 삭제)는 관리자만 가능하다. | 사용자 관리 API 호출 시 | 403 Forbidden 반환 |
| AUTH-R-016 | 용어사전 조회, 메일 분석 결과 조회, 서비스 상태 조회는 관리자 및 일반 사용자 모두 가능하다. | 조회 API 호출 시 | - |

#### 5. 민감정보 보호

| 규칙 번호 | 규칙 내용 | 적용 조건 | 위반 시 처리 |
|-----------|-----------|-----------|--------------|
| AUTH-R-017 | IMAP 계정 비밀번호(MAIL_PASSWORD)는 환경변수(.env.local)에만 저장하며, DB나 클라이언트 응답에 포함하지 않는다. | IMAP 설정 저장/조회 시 | 노출 발견 시 즉시 수정 |
| AUTH-R-018 | Claude API 키(ANTHROPIC_API_KEY)는 환경변수(.env.local)에만 저장하며, DB나 클라이언트 응답에 포함하지 않는다. | API 키 사용 시 | 노출 발견 시 즉시 수정 |
| AUTH-R-019 | iron-session 암호화 비밀키는 환경변수로 관리하며, 32자 이상의 랜덤 문자열을 사용한다. | 세션 암호화 시 | - |
| AUTH-R-020 | 환경설정 조회 API에서 비밀번호/API 키 필드는 마스킹 처리하여 반환한다 (예: "****" 또는 설정 여부만 boolean으로 반환). | 환경설정 조회 시 | 평문 반환 금지 |

### 예외 사항
- 로그인 페이지(`/login`)는 미인증 상태에서 접근 가능하다.
- 서버 헬스체크 엔드포인트(`/api/health`)는 인증 없이 접근 가능하다.

### 구현 가이드
- 세션 관리: iron-session 라이브러리를 사용하여 서버 사이드 암호화 쿠키 세션을 구현한다.
- 비밀번호 해싱: bcrypt 라이브러리를 사용한다.
- 미들웨어: Next.js App Router의 middleware.ts에서 인증 상태를 검증한다.
- 환경변수: `.env.local` 파일을 통해 민감정보를 관리하며, 이 파일은 .gitignore에 포함한다.

### 관련 문서
- 관련 데이터 정의: `docs/specs/datas/` (사용자 계정 테이블)
- 관련 화면 정의: `docs/specs/screens/` (로그인 페이지, 환경설정 페이지)
