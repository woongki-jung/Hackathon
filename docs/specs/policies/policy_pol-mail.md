# 메일 수신 정책 정의

## 개요
- IMAP 프로토콜 기반 메일 수신 및 임시 저장에 관한 비즈니스 규칙을 정의한다.
- 백그라운드에서 주기적으로 메일함을 확인하고 신규 메일을 텍스트로 저장하는 프로세스의 규칙을 명시한다.
- PRD의 "백엔드 기능 > 1. 메일 수신" 요구사항에 기반한다.

---

## POL-MAIL 메일 수신 정책

### 기본 정보
| 항목 | 내용 |
|------|------|
| 정책명 | 메일 수신 정책 |
| 분류 | 비즈니스 규칙 |
| 적용 범위 | 백그라운드 메일 수신 프로세스, IMAP 연결, 메일 임시 저장 |
| 우선순위 | 높음 |
| 관련 정책 | POL-AUTH (IMAP 인증 정보 보호), POL-DATA (임시 파일 저장 경로), POL-TERM (분석 대상 파일 제공) |

### 정책 규칙

#### 1. IMAP 연결

| 규칙 번호 | 규칙 내용 | 적용 조건 | 위반 시 처리 |
|-----------|-----------|-----------|--------------|
| MAIL-R-001 | IMAP 서버 연결 시 SSL/TLS 암호화를 사용한다. 기본 포트는 993(IMAPS)이다. | 메일 서버 연결 시 | 비암호화 연결 차단, 오류 로그 기록 |
| MAIL-R-002 | IMAP 접속 정보(호스트, 포트, 아이디, 비밀번호, SSL 여부)는 환경변수(MAIL_IMAP_HOST, MAIL_IMAP_PORT, MAIL_USERNAME, MAIL_PASSWORD, MAIL_USE_SSL)에서 읽는다. | 메일 수신 프로세스 시작 시 | 환경변수 미설정 시 메일 수신 프로세스 건너뜀, 경고 로그 기록 |
| MAIL-R-003 | IMAP 연결 실패 시 최대 3회까지 재시도하며, 재시도 간격은 10초로 한다. | IMAP 연결 실패 시 | 3회 실패 후 해당 주기 메일 확인 건너뜀, 오류 로그 기록 |

#### 2. 메일 확인 주기

| 규칙 번호 | 규칙 내용 | 적용 조건 | 위반 시 처리 |
|-----------|-----------|-----------|--------------|
| MAIL-R-004 | 메일 확인은 node-cron 스케줄러를 사용하여 주기적으로 실행한다. 기본 주기는 환경변수(MAIL_CHECK_INTERVAL)로 설정하며, 기본값은 3600000ms(1시간)이다. | 서버 실행 중 | - |
| MAIL-R-005 | 서버 시작 시 최초 1회 메일 확인을 즉시 실행한다. | 서버 시작 시 | - |
| MAIL-R-006 | 이전 메일 확인 작업이 아직 진행 중인 경우, 다음 주기의 메일 확인을 건너뛴다 (중복 실행 방지). | 스케줄러 트리거 시 | 건너뜀 로그 기록 |

#### 3. 메일 수신 범위

| 규칙 번호 | 규칙 내용 | 적용 조건 | 위반 시 처리 |
|-----------|-----------|-----------|--------------|
| MAIL-R-007 | INBOX 폴더의 UNSEEN(읽지 않은) 메일만 수신 대상으로 한다. | 메일 목록 조회 시 | - |
| MAIL-R-008 | 수신 처리 완료된 메일은 IMAP 서버에서 SEEN(읽음) 플래그를 설정한다. | 메일 텍스트 저장 성공 후 | 플래그 설정 실패 시 경고 로그 기록 (다음 주기에 재처리 가능) |
| MAIL-R-009 | 1회 수신 시 최대 50건까지 처리한다. 50건 초과분은 다음 주기에 처리한다. | 메일 목록 조회 결과가 50건 초과 시 | - |

#### 4. 메일 저장

| 규칙 번호 | 규칙 내용 | 적용 조건 | 위반 시 처리 |
|-----------|-----------|-----------|--------------|
| MAIL-R-010 | 수신된 메일의 제목과 본문(텍스트)을 추출하여 MAIL_STORAGE_PATH(기본값: `./data/mails`) 경로에 텍스트 파일로 저장한다. | 메일 수신 성공 시 | 저장 실패 시 오류 로그 기록, 해당 메일 SEEN 플래그 미설정 |
| MAIL-R-011 | 저장 파일명은 `{timestamp}_{messageId_hash}.txt` 형식을 사용한다. timestamp는 밀리초 단위 Unix 시간이다. | 메일 저장 시 | - |
| MAIL-R-012 | 메일 본문은 HTML인 경우 텍스트로 변환하여 저장한다. 텍스트 변환 시 HTML 태그를 제거하고 본문 텍스트만 추출한다. | HTML 형식 메일 수신 시 | - |
| MAIL-R-013 | 첨부파일은 저장 대상에서 제외한다. 메일 제목과 본문 텍스트만 저장한다. | 첨부파일이 있는 메일 수신 시 | - |

#### 5. 실행 상태 기록

| 규칙 번호 | 규칙 내용 | 적용 조건 | 위반 시 처리 |
|-----------|-----------|-----------|--------------|
| MAIL-R-014 | 메일 확인 작업의 마지막 실행 시점과 실행 결과(성공/실패, 수신 건수)를 DB에 기록한다. | 매 메일 확인 작업 완료 시 | - |
| MAIL-R-015 | 실행 상태 정보는 프론트엔드 "서비스 상태 조회" 페이지에서 조회할 수 있어야 한다. | 서비스 상태 조회 API 호출 시 | - |

### 예외 사항
- IMAP 접속 정보(환경변수)가 설정되지 않은 경우, 메일 수신 프로세스는 시작하지 않고 서비스 상태를 "미설정"으로 표시한다.
- 네트워크 장애로 IMAP 연결이 불가능한 경우, 해당 주기를 건너뛰고 다음 주기에 재시도한다.

### 구현 가이드
- IMAP 클라이언트: imapflow 라이브러리를 사용한다.
- 스케줄러: node-cron을 사용하여 Next.js 서버 프로세스 내에서 백그라운드 작업을 실행한다.
- HTML-to-Text 변환: 적절한 HTML 파서 라이브러리를 사용하여 본문 텍스트를 추출한다.

### 관련 문서

- 관련 화면 정의: `docs/specs/screens/` (서비스 상태 조회 페이지)
