# POL-MAIL: 메일 수신 정책

## 정책 개요

| 항목 | 내용 |
|------|------|
| 정책 코드 | POL-MAIL |
| 정책명 | 메일 수신 정책 |
| 적용 대상 | 메일함 모니터링, 메일 읽기, 메일 내용 추출 |
| 관련 PRD 기능 | 메일 수신 기능 |
| 의존 정책 | POL-AUTH |

## 정책 상세

### MAIL-01: 메일함 모니터링 주기

- 메일함 확인 주기는 `MAIL_POLL_INTERVAL_SECONDS` 환경변수로 설정한다.
- 기본값: 60초
- 최소값: 30초 (Graph API Rate Limit 고려)
- 최대값: 3600초 (1시간)
- 범위를 벗어난 값이 설정된 경우 기본값(60초)을 적용하고 경고 로그를 출력한다.

### MAIL-02: 수신 대상 메일함

- 모니터링 대상 메일함은 `MAIL_MAILBOX_NAME` 환경변수로 지정한다.
- 기본값: `Inbox`
- 지정된 메일함이 존재하지 않을 경우 에러 로그를 기록하고 `Inbox`로 폴백한다.

### MAIL-03: 메일 필터링 규칙

- `MAIL_FETCH_UNREAD_ONLY`가 `true`인 경우 읽지 않은 메일만 조회한다 (기본값: `true`).
- 1회 조회 시 최대 `MAIL_MAX_FETCH_COUNT`건을 가져온다 (기본값: 50).
- 조회 순서: 수신 시간 내림차순 (최신 메일 우선).
- 이미 처리된 메일의 재처리를 방지하기 위해 처리 완료된 메일의 Message ID를 로컬에 기록한다.

### MAIL-04: 메일 처리 후 상태 변경

- 메일 내용을 성공적으로 저장한 후, 해당 메일을 "읽음(isRead: true)" 상태로 변경한다.
- 상태 변경 실패 시 로그에 기록하되, 다음 폴링에서 중복 처리될 수 있음을 감안한다.
- 메일을 삭제하거나 이동하지 않는다 (원본 보존 원칙).

### MAIL-05: 메일 내용 추출 범위

- 추출 대상: 메일 제목(Subject)과 본문(Body)
- 본문 형식: HTML 본문인 경우 태그를 제거하고 텍스트만 추출한다.
- 첨부파일은 처리 대상에서 제외한다.
- 메일 헤더 정보(발신자, 수신일시)는 메타데이터로 함께 저장한다.

### MAIL-06: 오류 처리 정책

- Graph API 호출 실패 시:
  - HTTP 429 (Rate Limit): `Retry-After` 헤더 값만큼 대기 후 재시도
  - HTTP 401/403 (인증 실패): 토큰 갱신 후 재시도 (POL-AUTH 참조)
  - HTTP 5xx (서버 오류): 30초 대기 후 최대 3회 재시도
  - 네트워크 오류: 다음 폴링 주기까지 대기
- 연속 5회 이상 실패 시 에러 알림을 트레이 아이콘 풍선 알림으로 표시한다.

## 제약 사항

- Microsoft Graph API Rate Limit을 준수한다 (테넌트당 분당 10,000 요청).
- 대용량 메일(본문 1MB 초과)은 처음 1MB까지만 추출하고 잘린 사실을 메타데이터에 기록한다.
