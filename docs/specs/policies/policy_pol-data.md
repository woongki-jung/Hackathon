# 데이터 저장 정책 정의

## 개요
- SQLite 데이터베이스 및 파일 시스템 기반 데이터 저장, 보존, 삭제에 관한 정책을 정의한다.
- 모든 데이터는 서버 로컬 파일 시스템의 `./data/` 디렉터리 하위에 저장한다.
- PRD의 데이터 저장 요구사항 및 CLAUDE.local.md의 경로 설정에 기반한다.

---

## POL-DATA 데이터 저장 정책

### 기본 정보
| 항목 | 내용 |
|------|------|
| 정책명 | 데이터 저장 정책 |
| 분류 | 데이터 |
| 적용 범위 | SQLite DB, 메일 임시 저장 파일, 용어 해설집 파일, 환경변수 파일 |
| 우선순위 | 높음 |
| 관련 정책 | POL-AUTH (사용자 계정 데이터), POL-MAIL (메일 임시 파일), POL-TERM (용어 해설집 파일) |

### 정책 규칙

#### 1. 데이터베이스 관리

| 규칙 번호 | 규칙 내용 | 적용 조건 | 위반 시 처리 |
|-----------|-----------|-----------|--------------|
| DATA-R-001 | SQLite 데이터베이스 파일은 환경변수 DATABASE_PATH(기본값: `./data/app.db`)에 저장한다. | 서버 시작 시 | 경로가 존재하지 않으면 자동 생성 |
| DATA-R-002 | 데이터베이스 접근은 better-sqlite3 드라이버와 Drizzle ORM을 통해서만 수행한다. 직접 SQL 파일 조작을 금지한다. | 모든 DB 접근 시 | - |
| DATA-R-003 | 서버 시작 시 Drizzle ORM의 마이그레이션을 자동 실행하여 스키마를 최신 상태로 유지한다. | 서버 시작 시 | 마이그레이션 실패 시 서버 시작 중단, 오류 로그 기록 |
| DATA-R-004 | DB 파일이 존재하지 않는 경우 자동으로 생성한다. | 최초 서버 시작 시 | - |

#### 2. 파일 시스템 저장

| 규칙 번호 | 규칙 내용 | 적용 조건 | 위반 시 처리 |
|-----------|-----------|-----------|--------------|
| DATA-R-005 | 메일 임시 파일 저장 경로는 환경변수 MAIL_STORAGE_PATH(기본값: `./data/mails`)를 사용한다. | 메일 수신 시 | 경로가 존재하지 않으면 자동 생성 |
| DATA-R-006 | 용어 해설집 파일 저장 경로는 환경변수 GLOSSARY_STORAGE_PATH(기본값: `./data/terms`)를 사용한다. | 용어 분석 완료 시 | 경로가 존재하지 않으면 자동 생성 |
| DATA-R-007 | 모든 데이터 저장 경로는 `./data/` 디렉터리 하위에 위치해야 한다. 상위 디렉터리 탈출(path traversal)을 허용하지 않는다. | 파일 읽기/쓰기 시 | 경로 검증 실패 시 작업 거부, 오류 로그 기록 |

#### 3. 메일 임시 파일 보존/삭제

| 규칙 번호 | 규칙 내용 | 적용 조건 | 위반 시 처리 |
|-----------|-----------|-----------|--------------|
| DATA-R-008 | 메일 임시 파일은 용어 분석 처리가 완료된 후에도 즉시 삭제하지 않고 보존한다. | 용어 분석 완료 후 | - |
| DATA-R-009 | 메일 임시 파일의 보존 기간은 30일로 한다. 30일이 경과한 파일은 다음 메일 확인 주기에 자동 삭제한다. | 메일 확인 주기 실행 시 | 삭제 실패 시 경고 로그 기록, 다음 주기에 재시도 |
| DATA-R-010 | 메일 임시 파일 삭제는 물리적 삭제(하드 삭제)로 수행한다. | 보존 기간 경과 시 | - |

#### 6. 환경변수 관리

### 예외 사항
- 개발 환경에서는 `./data/` 경로 대신 임시 경로를 사용할 수 있으나, 환경변수로 명시적으로 설정해야 한다.
- 데이터베이스 백업은 본 정책의 범위에 포함하지 않으며, 운영 환경에서 별도로 관리한다.

### 구현 가이드
- DB 드라이버: better-sqlite3를 사용하여 동기식 SQLite 접근을 수행한다.
- ORM: Drizzle ORM을 사용하여 스키마 정의 및 쿼리를 수행한다.
- 파일 I/O: Node.js fs 모듈을 사용하며, 비동기 방식(fs/promises)을 권장한다.
- 경로 관리: path.resolve()를 사용하여 상대 경로를 절대 경로로 변환하고, path traversal을 방지한다.
