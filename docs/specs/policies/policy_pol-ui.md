# POL-UI: UI/UX 정책

## 정책 개요

| 항목 | 내용 |
|------|------|
| 정책 코드 | POL-UI |
| 정책명 | UI/UX 정책 |
| 적용 대상 | 윈도우 트레이 아이콘, 환경설정 화면, 용어사전 뷰어 |
| 관련 PRD 기능 | 윈도우 트레이 기능, 환경설정 기능, 용어사전 뷰어 |
| 의존 정책 | POL-TERM (사전 데이터 표시) |

## 정책 상세

### UI-01: 윈도우 트레이 아이콘 동작 규칙

- 앱 실행 시 시스템 트레이 영역에 아이콘을 표시한다.
- 트레이 아이콘 좌클릭: 메인 윈도우(용어사전 뷰어)를 표시하거나 포커스한다.
- 트레이 아이콘 우클릭: 컨텍스트 메뉴를 표시한다.
  - 메뉴 항목: "열기", "환경설정", "메일 즉시 확인", 구분선, "종료"
- 메인 윈도우 닫기(X 버튼) 클릭 시 앱을 종료하지 않고 트레이로 최소화한다.
- 앱 종료는 트레이 메뉴의 "종료" 항목을 통해서만 가능하다.

### UI-02: 트레이 알림 정책

- 다음 이벤트 발생 시 트레이 풍선 알림(Balloon Notification)을 표시한다:
  - 신규 메일 수신 및 분석 완료: "{n}건의 새로운 용어가 추가되었습니다"
  - 메일 수신 오류 연속 5회 이상: "메일 서버 연결에 문제가 있습니다"
  - 디스크 용량 부족: "저장 공간이 부족합니다"
- 알림 표시 시간: 5초
- 알림 클릭 시 메인 윈도우를 표시한다.

### UI-03: 환경설정 화면 구성

- 환경설정은 별도 다이얼로그 창으로 제공한다.
- 설정 항목 그룹:
  - **메일 계정**: 이메일 주소 표시 (Azure AD 인증은 환경변수로 관리하므로, 계정 연결 상태만 표시)
  - **수신 설정**: 폴링 주기(초), 읽지 않은 메일만 조회 여부, 1회 최대 조회 수
  - **저장 경로**: 분석 요청 폴더 경로 (폴더 선택 다이얼로그 지원)
  - **용어 분석**: 일일 최대 API 호출 횟수, 불용어 목록 편집
- "저장" 버튼 클릭 시 설정을 적용하고 다이얼로그를 닫는다.
- "취소" 버튼 클릭 시 변경 사항을 폐기하고 다이얼로그를 닫는다.
- 설정 변경 즉시 반영 (앱 재시작 불필요).

### UI-04: 용어사전 뷰어 메인 화면

- 메인 화면 상단에 검색창을 중앙 배치한다 (구글 검색창 형태).
- 검색창 아래에 "최근 빈도 높은 용어" 섹션을 표시한다.
  - 최근 7일간 `source_count` 증가량 기준 상위 10개 용어를 바로가기(태그/칩) 형태로 표시.
  - 바로가기 클릭 시 해당 용어의 상세 해설을 표시한다.
- 화면 하단에 앱 상태 정보를 표시한다:
  - "총 {n}개 용어 등록됨 | 마지막 메일 확인: {시각}"

### UI-05: 검색 동작 규칙

- 검색 대상: 용어 원문(`term`) 및 해설(`description`)
- 검색 방식: 부분 일치 (입력 문자열이 포함된 모든 항목)
- 한글/영문 대소문자 구분 없이 검색한다.
- 검색 결과는 관련도순(용어 원문 정확 일치 > 용어 부분 일치 > 해설 포함)으로 정렬한다.
- 검색어 입력 시 300ms 디바운스를 적용하여 실시간 검색 결과를 표시한다.
- 검색 결과가 없는 경우 "일치하는 용어가 없습니다" 메시지를 표시한다.

### UI-06: 용어 상세 표시 규칙

- 검색 결과 목록에서 용어를 선택하면 상세 정보를 표시한다.
- 상세 정보 표시 항목:
  - 용어 원문
  - 카테고리 (EMR / Business / Abbreviation) - 카테고리별 색상 구분
  - 1줄 요약
  - 상세 설명
  - 관련 용어 (클릭 시 해당 용어로 이동)
  - 발견 횟수 및 최초/최근 발견 일시
- 상세 화면에서 "목록으로" 버튼으로 검색 결과 목록으로 복귀할 수 있다.

### UI-07: 윈도우 크기 및 레이아웃

- 기본 윈도우 크기: 800 x 600 픽셀
- 최소 윈도우 크기: 640 x 480 픽셀
- 사용자가 조정한 윈도우 크기와 위치는 다음 실행 시 복원한다.
- UI 프레임워크: WinUI 3 (Windows App SDK) 또는 WPF (.NET 10 호환)

## 제약 사항

- Windows 11 이상에서만 동작을 보장한다.
- 고DPI(4K) 디스플레이에서의 스케일링을 지원해야 한다.
- 다크 모드/라이트 모드는 시스템 설정을 따른다.
- 키보드 접근성: Tab 키로 모든 UI 요소를 탐색할 수 있어야 한다.
