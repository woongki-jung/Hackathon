# 개인정보 필터링 기능 정의

## 개요
- 메일 본문 및 추출 결과에서 개인정보 패턴을 탐지하고 마스킹하는 기능을 정의한다.
- 적용 범위: Claude API 호출 전 메일 본문 전처리, 분석 결과 후처리

---

## TERM-PII-001 개인정보 필터링

### 기본 정보
| 항목 | 내용 |
|------|------|
| 기능명 | 개인정보 필터링 |
| 분류 | 도메인 특화 로직 |
| 레이어 | lib/analysis |
| 트리거 | TERM-BATCH-001에서 분석 전 전처리로 호출 |
| 관련 정책 | POL-AUTH (민감정보 보호 원칙) |

### 입력 / 출력

#### maskPiiInText

##### 입력 (Input)
| 파라미터 | 타입 | 필수 | 설명 | 유효성 규칙 |
|----------|------|------|------|-------------|
| text | string | ✅ | 원본 텍스트 (메일 본문) | - |

##### 출력 (Output)
| 항목 | 타입 | 설명 |
|------|------|------|
| maskedText | string | 개인정보가 마스킹된 텍스트 |
| piiFound | number | 탐지된 개인정보 패턴 수 |

### 탐지 대상 패턴

| 패턴 | 정규식 예시 | 마스킹 결과 |
|------|------------|-------------|
| 이메일 주소 | `/[\w.-]+@[\w.-]+\.\w+/` | `[EMAIL]` |
| 전화번호 (한국) | `/01[0-9]-?\d{3,4}-?\d{4}/` | `[PHONE]` |
| 주민등록번호 | `/\d{6}-?[1-4]\d{6}/` | `[SSN]` |
| 신용카드 번호 | `/\d{4}-?\d{4}-?\d{4}-?\d{4}/` | `[CARD]` |
| 계좌번호 패턴 | `/\d{3,6}-\d{2,6}-\d{4,8}/` | `[ACCOUNT]` |

### 처리 흐름

1. **패턴 목록 로드**: 사전 정의된 정규식 패턴 목록
2. **순차 적용**: 각 패턴을 텍스트에 순차 적용하여 마스킹
3. **카운트**: 탐지된 패턴 수 합산
4. **결과 반환**: 마스킹된 텍스트 + 탐지 수

### 구현 가이드

- **패턴**: Pure Function - lib/analysis/pii-filter.ts
- **정규식 관리**: 패턴 목록을 상수로 분리하여 유지보수
- **성능**: 정규식 사전 컴파일 (RegExp 인스턴스 재사용)
- **한계**: 정규식 기반 탐지의 한계를 인지하고, 완벽한 필터링보다는 최선의 노력(best-effort)으로 접근
- **외부 의존성**: 없음 (순수 유틸리티)

### 관련 기능
- **이 기능을 호출하는 기능**: TERM-BATCH-001
- **이 기능이 호출하는 기능**: 없음

### 테스트 시나리오

| 시나리오 | 입력 조건 | 기대 결과 |
|----------|-----------|-----------|
| 이메일 마스킹 | "연락처: user@test.com" | "연락처: [EMAIL]" |
| 전화번호 마스킹 | "010-1234-5678" | "[PHONE]" |
| 주민번호 마스킹 | "900101-1234567" | "[SSN]" |
| 복합 패턴 | 이메일+전화 포함 텍스트 | 모두 마스킹, piiFound=2 |
| 개인정보 없음 | 일반 업무 메일 | 원본 그대로, piiFound=0 |
