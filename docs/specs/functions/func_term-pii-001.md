# 개인정보 필터링 기능 정의

## 개요

- **기능 목적**: 해설 생성 전에 텍스트에서 환자 개인정보(이름, 주민번호 등)를 탐지하여 제거 또는 마스킹한다.
- **적용 범위**: 배치 분석(TERM-BATCH-001)에서 용어 추출 전 텍스트 전처리 단계.

---

## TERM-PII-001: 개인정보 필터링

### 기본 정보

| 항목 | 내용 |
|------|------|
| 기능명 | 개인정보 필터링 |
| 분류 | 도메인 특화 로직 |
| 레이어 | Domain |
| 트리거 | TERM-BATCH-001에서 파일 파싱 후, 용어 추출 전 |
| 관련 정책 | POL-TERM (제약사항: 환자 개인정보 필터링) |

### 입력 / 출력

#### 입력 (Input)

| 파라미터 | 타입 | 필수 | 설명 | 유효성 규칙 |
|----------|------|------|------|-------------|
| text | string | ✅ | 필터링 대상 텍스트 | |

#### 출력 (Output)

| 항목 | 타입 | 설명 |
|------|------|------|
| sanitizedText | string | 개인정보가 제거/마스킹된 텍스트 |
| detectedCount | int | 탐지된 개인정보 항목 수 |

### 처리 흐름

1. **주민등록번호 패턴 탐지**: `\d{6}[-]?\d{7}` 패턴을 탐지하여 `[개인정보 제거됨]`으로 치환한다.
2. **전화번호 패턴 탐지**: `01[016789]-?\d{3,4}-?\d{4}` 패턴을 탐지하여 마스킹한다.
3. **이메일 주소 패턴**: 이메일은 메타데이터(발신자)에는 포함되나, 본문 내 환자 관련 이메일은 마스킹한다.
4. **한글 이름 패턴**: 2~4글자 한글 이름은 단독으로 판별이 어려우므로, "환자명:", "성명:" 등 선행 키워드와 결합된 패턴만 탐지한다.
5. **결과 반환**: 마스킹된 텍스트와 탐지 건수를 반환한다.

### 구현 가이드

- **패턴**: 순수 함수. 정규식 패턴 목록은 설정으로 관리하여 확장 가능.
- **보안**: 탐지된 개인정보는 로그에도 출력하지 않는다. 건수만 기록.
- **한계**: 완벽한 개인정보 탐지는 불가능하므로, 명확한 패턴만 처리하고 나머지는 운영 가이드로 보완.

### 관련 기능

- **이 기능을 호출하는 기능**: TERM-BATCH-001
- **이 기능이 호출하는 기능**: 없음 (순수 함수)

### 테스트 시나리오

| 시나리오 | 입력 조건 | 기대 결과 |
|----------|-----------|-----------|
| 주민번호 탐지 | "환자 901215-1234567" | "환자 [개인정보 제거됨]", detectedCount=1 |
| 전화번호 탐지 | "연락처 010-1234-5678" | 마스킹 처리, detectedCount=1 |
| 이름+키워드 탐지 | "환자명: 홍길동" | "환자명: [개인정보 제거됨]" |
| 일반 텍스트 | "EMR 시스템 연동" | 변경 없음, detectedCount=0 |
| 복합 케이스 | 주민번호+전화번호 포함 | 모두 마스킹, detectedCount=2 |
