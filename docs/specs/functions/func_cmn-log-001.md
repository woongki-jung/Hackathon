# 로깅 및 민감정보 마스킹 기능 정의

## 개요

- **기능 목적**: 애플리케이션 전반의 구조화 로깅을 제공하며, 인증 정보 등 민감 데이터를 자동으로 마스킹한다.
- **적용 범위**: 모든 기능에서 횡단적으로 사용한다.

### 입력 / 출력

#### 입력 (Input)

| 파라미터 | 타입 | 필수 | 설명 | 유효성 규칙 |
|----------|------|------|------|-------------|
| level | enum | ✅ | 로그 레벨 (Debug/Info/Warning/Error/Critical) | |
| message | string | ✅ | 로그 메시지 | |
| context | object? | | 구조화 로그 컨텍스트 (키-값 쌍) | |
| exception | Exception? | | 예외 객체 | |

#### 출력 (Output)

없음 (부수 효과로 로그 출력)

### 처리 흐름

1. **민감정보 마스킹**: 메시지와 컨텍스트에서 마스킹 대상 패턴을 검출하여 `***`로 치환한다.
   - 마스킹 대상: `client_secret`, `access_token`, `api_key`, `password` 등의 키 값
   - Bearer 토큰 패턴: `Bearer ey...` -> `Bearer ***`
   - API 키 패턴: `sk-ant-...` -> `sk-ant-***`
2. **로그 포맷팅**: 타임스탬프, 레벨, 메시지, 컨텍스트를 구조화 형식으로 조합한다.
3. **출력**: 설정된 로그 싱크(파일, 콘솔 등)에 기록한다.

### 구현 가이드

- **패턴**: `ILogger` 인터페이스를 통한 추상화. 로그 프레임워크 선택은 개발자에게 위임 (Serilog, NLog, Microsoft.Extensions.Logging 등).
- **성능**: 로그 레벨 필터링을 먼저 수행하여 불필요한 마스킹 처리를 방지한다.
- **보안**: 마스킹은 로그 파이프라인의 최종 단계에서 적용하여 누락을 방지한다.
