# 분석 요청 파일 생성 기능 정의

## 개요

- **기능 목적**: 추출된 메일 내용을 정해진 명명 규칙과 파일 형식에 따라 분석 요청 폴더에 텍스트 파일로 저장한다.
- **적용 범위**: 메일 내용 추출(MAIL-PROC-001) 후 호출된다.

---

## DATA-FILE-001: 분석 요청 파일 생성

### 기본 정보

| 항목 | 내용 |
|------|------|
| 기능명 | 분석 요청 파일 생성 |
| 분류 | 도메인 특화 로직 |
| 레이어 | Infrastructure |
| 트리거 | MAIL-PROC-001에서 추출한 메일 데이터 저장 요청 |
| 관련 정책 | POL-DATA (DATA-01, DATA-02, DATA-03) |

### 입력 / 출력

#### 입력 (Input)

| 파라미터 | 타입 | 필수 | 설명 | 유효성 규칙 |
|----------|------|------|------|-------------|
| subject | string | ✅ | 메일 제목 | |
| plainTextBody | string | ✅ | 텍스트 본문 | |
| fromAddress | string | ✅ | 발신자 이메일 | |
| receivedAt | DateTime | ✅ | 수신 일시 | |
| messageId | string | ✅ | Message ID | 비어 있지 않은 문자열 |

#### 출력 (Output)

| 항목 | 타입 | 설명 |
|------|------|------|
| filePath | string | 생성된 파일의 전체 경로 |
| skipped | boolean | 동일 파일 존재로 건너뛴 경우 true |

#### 예외 / 오류

| 조건 | 오류 코드 | 설명 |
|------|-----------|------|
| 디렉터리 접근 불가 | ERR_DATA_DIR_ACCESS | 분석 요청 폴더 접근 권한 없음 |
| 파일 쓰기 실패 | ERR_DATA_WRITE_FAILED | 파일 저장 실패 |
| 디스크 용량 부족 | ERR_DATA_DISK_LOW | 1GB 미만 (CMN-FS-001) |

### 처리 흐름

1. **디스크 용량 확인**: CMN-FS-001로 용량을 확인한다. 1GB 미만 시 저장을 중단한다 (POL-DATA 제약사항).
2. **디렉터리 보장**: OUTPUT_ANALYSIS_DIR 디렉터리를 확인/생성한다 (DATA-01).
3. **파일명 생성**: 명명 규칙에 따라 파일명을 생성한다 (DATA-02).
   - 수신일시: `yyyyMMdd_HHmmss` 형식으로 변환
   - MessageID 해시: SHA256 해시의 앞 8자리 추출
   - 파일명: `{수신일시}_{해시}.txt`
4. **중복 확인**: 동일 파일명이 이미 존재하면 건너뛴다 (DATA-02).
5. **파일 내용 구성**: YAML frontmatter + 본문 형식으로 조합한다 (DATA-03).
   ```
   ---
   subject: {메일 제목}
   from: {발신자 이메일}
   received_at: {수신 일시 ISO 8601}
   message_id: {메일 Message ID}
   ---

   {메일 본문 텍스트}
   ```
6. **파일 저장**: UTF-8 인코딩(BOM 없음)으로 저장한다 (DATA-03).
7. **결과 반환**: 파일 경로와 건너뜀 여부를 반환한다.

### 구현 가이드

- **패턴**: 순수 함수로 파일명 생성 로직 분리. 파일 I/O는 CMN-FS-001 활용.
- **보안**: 메일 제목에 파일명에 사용 불가한 문자가 포함될 수 있으나, 파일명에는 해시를 사용하므로 문제 없음.
- **외부 의존성**: SHA256 해시, 파일 시스템

### 관련 기능

- **이 기능을 호출하는 기능**: MAIL-RECV-001, TERM-BATCH-001 (파일 경로 참조)
- **이 기능이 호출하는 기능**: CMN-FS-001, CMN-LOG-001

### 테스트 시나리오

| 시나리오 | 입력 조건 | 기대 결과 |
|----------|-----------|-----------|
| 정상 생성 | 유효한 메일 데이터 | 파일 생성, skipped=false |
| 파일명 형식 | 수신일시 2026-03-13 14:30:22, ID 해시 a1b2c3d4 | "20260313_143022_a1b2c3d4.txt" |
| 중복 건너뜀 | 동일 파일명 존재 | skipped=true |
| YAML frontmatter | 메타데이터 포함 | 파일 앞부분에 --- 구분 YAML 메타데이터 |
| UTF-8 BOM 없음 | 한글 포함 메일 | BOM 없는 UTF-8 인코딩 |
| 디스크 부족 | 500MB 남음 | ERR_DATA_DISK_LOW |
| 한글 경로 | OUTPUT_ANALYSIS_DIR에 한글 포함 | 정상 동작 |
