# 불용어 필터링 기능 정의

## 개요

- **기능 목적**: 추출된 용어 후보에서 일반적인 영어 약어 및 비전문 용어를 불용어 목록 기반으로 필터링한다.
- **적용 범위**: 용어 추출(TERM-EXT-001) 결과에 대해 호출된다.

---

## TERM-EXT-002: 불용어 필터링

### 기본 정보

| 항목 | 내용 |
|------|------|
| 기능명 | 불용어 필터링 |
| 분류 | 도메인 특화 로직 |
| 레이어 | Domain |
| 트리거 | TERM-EXT-001 용어 추출 후 |
| 관련 정책 | POL-TERM (TERM-02) |

### 입력 / 출력

#### 입력 (Input)

| 파라미터 | 타입 | 필수 | 설명 | 유효성 규칙 |
|----------|------|------|------|-------------|
| candidates | TermCandidate[] | ✅ | 필터링 대상 용어 후보 목록 | |
| stopWords | string[] | ✅ | 불용어 목록 | |

#### 출력 (Output)

| 항목 | 타입 | 설명 |
|------|------|------|
| filtered | TermCandidate[] | 불용어가 제거된 용어 목록 |
| removedCount | int | 제거된 용어 수 |

### 처리 흐름

1. **불용어 목록 로드**: 설정 파일에서 불용어 목록을 가져온다 (CMN-CFG-001 경유).
   - 기본 불용어 예시: IT, OK, PM, AM, HR, PC, TV, US, UK, EU 등 (TERM-02)
2. **대소문자 무시 비교**: 불용어 매칭 시 대소문자를 구분하지 않는다.
3. **필터링**: candidates에서 불용어에 해당하는 항목을 제거한다.
4. **결과 반환**: 필터링된 목록과 제거 건수를 반환한다.

### 구현 가이드

- **패턴**: 순수 함수. 불용어 목록은 HashSet으로 관리하여 O(1) 조회.
- **확장성**: 불용어 목록은 사용자가 환경설정에서 편집 가능 (POL-UI UI-03, TERM-02).

### 관련 기능

- **이 기능을 호출하는 기능**: TERM-EXT-001
- **이 기능이 호출하는 기능**: 없음 (순수 함수)

### 테스트 시나리오

| 시나리오 | 입력 조건 | 기대 결과 |
|----------|-----------|-----------|
| 불용어 제거 | ["EMR", "OK", "PACS"], 불용어=["OK"] | ["EMR", "PACS"], removedCount=1 |
| 대소문자 무시 | ["ok"], 불용어=["OK"] | [], removedCount=1 |
| 빈 불용어 목록 | 후보 5개, 불용어=[] | 5개 모두 유지 |
| 전체 필터링 | ["IT", "OK"], 불용어=["IT", "OK"] | [], removedCount=2 |
