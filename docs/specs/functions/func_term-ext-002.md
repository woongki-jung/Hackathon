# 불용어 필터링 기능 정의

## 개요
- 추출된 용어 목록에서 불용어(stop_words)를 제거하는 기능을 정의한다.
- 적용 범위: 용어 추출 결과 후처리

---

## TERM-EXT-002 불용어 필터링

### 기본 정보
| 항목 | 내용 |
|------|------|
| 기능명 | 불용어 필터링 |
| 분류 | 도메인 특화 로직 |
| 레이어 | lib/analysis |
| 트리거 | TERM-EXT-001 용어 추출 후 |
| 관련 정책 | POL-TERM |

### 입력 / 출력

#### 1. 불용어 목록 조회 (getStopWords)

##### 출력 (Output)
| 항목 | 타입 | 설명 |
|------|------|------|
| words | string[] | 소문자 정규화된 불용어 목록 |

#### 2. 불용어 필터링 (filterStopWords)

##### 입력 (Input)
| 파라미터 | 타입 | 필수 | 설명 | 유효성 규칙 |
|----------|------|------|------|-------------|
| terms | ExtractedTerm[] | ✅ | 추출된 용어 목록 | - |
| stopWords | string[] | ✅ | 불용어 목록 | 소문자 정규화됨 |

##### 출력 (Output)
| 항목 | 타입 | 설명 |
|------|------|------|
| filtered | ExtractedTerm[] | 불용어가 제거된 용어 목록 |
| removedCount | number | 제거된 용어 수 |

### 처리 흐름

1. **불용어 목록 조회**: DB stop_words 테이블에서 전체 목록 조회
2. **Set 생성**: 불용어를 Set<string>으로 변환 (빠른 조회용)
3. **필터링**: 각 용어의 name을 소문자로 변환 후 Set에 포함 여부 확인
4. **결과 반환**: 불용어가 아닌 용어만 반환

### 구현 가이드

- **패턴**: Pure Function + Repository 조회 - lib/analysis/stop-word-filter.ts
- **성능**: 불용어 목록을 메모리 캐시하여 매 분석 시 DB 조회 최소화 (서버 시작 시 1회 로드)
- **소문자 정규화**: 비교 시 양쪽 모두 toLowerCase() 적용
- **외부 의존성**: Drizzle ORM

### 관련 기능
- **이 기능을 호출하는 기능**: TERM-EXT-001
- **이 기능이 호출하는 기능**: 없음

### 관련 데이터
- DATA-006 StopWord (stop_words 테이블)

### 테스트 시나리오

| 시나리오 | 입력 조건 | 기대 결과 |
|----------|-----------|-----------|
| 불용어 제거 | terms에 "the" 포함, stopWords에 "the" 있음 | "the" 제거 |
| 대소문자 무시 | terms에 "The", stopWords에 "the" | "The" 제거 |
| 불용어 없음 | 모두 유효한 용어 | 전체 반환, removedCount=0 |
| 빈 목록 | terms 빈 배열 | 빈 배열 반환 |
| 한국어 불용어 | terms에 "안녕하세요", stopWords에 "안녕하세요" | 해당 용어 제거 |
