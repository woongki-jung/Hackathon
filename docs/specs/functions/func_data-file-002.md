# 처리 완료 파일 관리 기능 정의

## 개요

- **기능 목적**: 분석이 완료된 파일을 Processed 디렉터리로 이동하여 재처리를 방지하고 파일을 정리한다.
- **적용 범위**: TERM-BATCH-001에서 파일 분석 완료 후 호출된다.

---

## DATA-FILE-002: 처리 완료 파일 관리

### 기본 정보

| 항목 | 내용 |
|------|------|
| 기능명 | 처리 완료 파일 관리 |
| 분류 | 도메인 특화 로직 |
| 레이어 | Infrastructure |
| 트리거 | 배치 분석(TERM-BATCH-001)에서 파일 처리 완료 시 |
| 관련 정책 | POL-DATA (DATA-06) |

### 입력 / 출력

#### 입력 (Input)

| 파라미터 | 타입 | 필수 | 설명 | 유효성 규칙 |
|----------|------|------|------|-------------|
| filePath | string | ✅ | 처리 완료된 파일 경로 | 존재하는 파일 |
| analysisDir | string | ✅ | 분석 요청 폴더 경로 | 유효한 디렉터리 |

#### 출력 (Output)

| 항목 | 타입 | 설명 |
|------|------|------|
| moved | boolean | 파일 이동 성공 여부 |
| destinationPath | string | 이동된 경로 (성공 시) |

#### 예외 / 오류

| 조건 | 오류 코드 | 설명 |
|------|-----------|------|
| 이동 실패 | ERR_DATA_MOVE_FAILED | 파일 이동 중 오류 |

### 처리 흐름

1. **Processed 디렉터리 보장**: `{analysisDir}/Processed/` 디렉터리를 확인/생성한다.
2. **파일 이동**: 원본 파일을 Processed 디렉터리로 이동한다 (DATA-06).
3. **이동 실패 시 폴백**: 파일을 원래 위치에 유지하고, 처리 완료 상태를 메타 파일(`.processed` 마커)에 기록한다 (DATA-06).
4. **결과 반환**: 이동 성공 여부와 대상 경로를 반환한다.

### 구현 가이드

- **패턴**: CMN-FS-001의 MoveFile, EnsureDirectory를 활용.
- **동시성**: 배치 처리 중 파일별로 순차 이동.

### 관련 기능

- **이 기능을 호출하는 기능**: TERM-BATCH-001
- **이 기능이 호출하는 기능**: CMN-FS-001, CMN-LOG-001

### 테스트 시나리오

| 시나리오 | 입력 조건 | 기대 결과 |
|----------|-----------|-----------|
| 정상 이동 | 유효한 파일 경로 | Processed로 이동, moved=true |
| 이동 실패 폴백 | 파일 잠금 중 | 원본 유지, .processed 마커 생성 |
| Processed 디렉터리 생성 | 디렉터리 미존재 | 자동 생성 후 이동 |
