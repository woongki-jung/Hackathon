# 환경설정 관리 기능 정의

## 개요

- **기능 목적**: 애플리케이션의 설정값을 읽기/쓰기/유효성 검사하고, 환경변수로부터의 폴백을 지원한다.
- **적용 범위**: 메일 수신(MAIL-RECV-001), 배치 처리(TERM-BATCH-001), UI 환경설정 화면(POL-UI UI-03) 등 설정값을 참조하는 모든 기능에서 사용한다.

---

## CMN-CFG-001: 환경설정 관리

### 기본 정보

| 항목 | 내용 |
|------|------|
| 기능명 | 환경설정 관리 |
| 분류 | 공통 기능 |
| 레이어 | Infrastructure |
| 트리거 | 앱 시작 시 로드, 설정 화면에서 저장 시, 설정값 참조 시 |
| 관련 정책 | POL-MAIL (MAIL-01~03), POL-DATA (DATA-01), POL-TERM (TERM-05 제약사항) |

### 입력 / 출력

#### 설정 로드 (Load)

**출력**

| 항목 | 타입 | 설명 |
|------|------|------|
| config | AppConfiguration | 전체 설정 객체 |

#### 설정 저장 (Save)

**입력**

| 파라미터 | 타입 | 필수 | 설명 | 유효성 규칙 |
|----------|------|------|------|-------------|
| mailUserEmail | string | ✅ | 메일 계정 이메일 | 이메일 형식 |
| pollIntervalSeconds | int | ✅ | 메일 확인 주기 | 30 이상 3600 이하 |
| fetchUnreadOnly | boolean | ✅ | 읽지 않은 메일만 조회 | |
| maxFetchCount | int | ✅ | 1회 최대 조회 수 | 1 이상 200 이하 |
| mailboxName | string | ✅ | 대상 메일함 이름 | 비어 있지 않은 문자열 |
| analysisDir | string | ✅ | 분석 요청 폴더 경로 | 유효한 디렉터리 경로 |
| maxDailyApiCalls | int | ✅ | 일일 최대 API 호출 | 1 이상 |
| stopWords | string[] | ✅ | 불용어 목록 | |

**출력**

| 항목 | 타입 | 설명 |
|------|------|------|
| success | boolean | 저장 성공 여부 |
| validationErrors | string[] | 유효성 검사 실패 항목 목록 |

#### 예외 / 오류

| 조건 | 오류 코드 | 설명 |
|------|-----------|------|
| 유효성 검사 실패 | ERR_CFG_VALIDATION | 설정값이 허용 범위를 벗어남 |
| 설정 파일 손상 | ERR_CFG_CORRUPTED | 설정 파일 파싱 불가 |
| 파일 쓰기 실패 | ERR_CFG_WRITE_FAILED | 설정 파일 저장 권한 없음 |

### 처리 흐름

**로드 흐름**:

1. **설정 파일 존재 확인**: 앱 디렉터리의 설정 파일(JSON)을 확인한다.
2. **파일 로드**: 존재하면 파싱하고, 없거나 손상되면 기본값을 사용한다.
3. **환경변수 오버라이드**: 대응하는 환경변수가 설정되어 있으면 해당 값으로 오버라이드한다.
4. **유효성 보정**: 범위를 벗어난 값은 기본값으로 보정하고 경고 로그를 출력한다 (MAIL-01).
   - pollIntervalSeconds: 30~3600 범위 벗어나면 60으로 보정
   - mailboxName: 존재하지 않으면 "Inbox"로 폴백 (MAIL-02)
5. **설정 객체 반환**: 검증된 설정 객체를 반환한다.

**저장 흐름**:

1. **유효성 검사**: 모든 입력값에 대해 허용 범위를 검사한다.
2. **실패 시 반환**: 유효성 검사 실패 항목을 목록으로 반환한다.
3. **성공 시 저장**: 설정 파일(JSON)에 저장한다. 민감 정보(인증 관련)는 CMN-SEC-001을 통해 별도 저장한다.
4. **변경 이벤트 발행**: 설정 변경을 구독자에게 알린다 (즉시 반영, UI-03).

### 구현 가이드

- **패턴**: Observer 패턴으로 설정 변경 시 구독자에게 알림. `IOptions<T>` 패턴과 유사한 구조.
- **성능**: 설정값은 메모리에 캐싱하여 빈번한 파일 I/O를 방지한다.
- **보안**: 인증 관련 설정(API 키 등)은 설정 파일에 포함하지 않고 CMN-SEC-001을 통해 관리한다.
- **외부 의존성**: JSON 직렬화 라이브러리, 파일 시스템

### 관련 기능

- **이 기능을 호출하는 기능**: MAIL-RECV-001, TERM-BATCH-001, TERM-EXT-002, VIEW-STATE-001
- **이 기능이 호출하는 기능**: CMN-SEC-001 (민감 설정 조회), CMN-LOG-001 (경고 로깅)

### 테스트 시나리오

| 시나리오 | 입력 조건 | 기대 결과 |
|----------|-----------|-----------|
| 정상 로드 | 유효한 설정 파일 존재 | 설정 객체 반환 |
| 파일 미존재 | 설정 파일 없음 | 기본값으로 설정 객체 반환 |
| 환경변수 오버라이드 | 파일 값 60, 환경변수 120 | 120 적용 |
| 범위 벗어남 보정 | pollIntervalSeconds=10 | 60으로 보정, 경고 로그 |
| 정상 저장 | 모든 값 유효 범위 내 | success=true |
| 유효성 실패 | pollIntervalSeconds=-1 | success=false, validationErrors 포함 |
| 설정 변경 알림 | 저장 성공 후 | 구독자에게 변경 이벤트 전달 |
| 파일 손상 | JSON 파싱 불가 | 기본값 적용, ERR_CFG_CORRUPTED 로그 |
