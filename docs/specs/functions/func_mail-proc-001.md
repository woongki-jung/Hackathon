# 메일 내용 추출 기능 정의

## 개요
- 수신된 메일의 제목과 본문 텍스트를 추출하고, HTML 형식인 경우 텍스트로 변환하는 기능을 정의한다.
- 적용 범위: 메일 수신 후 텍스트 파일 저장 전처리

---

## MAIL-PROC-001 메일 내용 추출

### 기본 정보
| 항목 | 내용 |
|------|------|
| 기능명 | 메일 내용 추출 |
| 분류 | 도메인 특화 로직 |
| 레이어 | lib/mail |
| 트리거 | MAIL-RECV-001에서 메일 수신 후 |
| 관련 정책 | POL-MAIL (MAIL-R-010, MAIL-R-012, MAIL-R-013) |

### 입력 / 출력

#### extractMailContent

##### 입력 (Input)
| 파라미터 | 타입 | 필수 | 설명 | 유효성 규칙 |
|----------|------|------|------|-------------|
| rawMail | RawMail | ✅ | IMAP에서 수신한 원시 메일 데이터 | - |

##### 출력 (Output)
| 항목 | 타입 | 설명 |
|------|------|------|
| subject | string | 메일 제목 |
| textBody | string | 텍스트로 변환된 본문 |
| receivedAt | string | 수신 일시 (ISO 8601) |
| messageIdHash | string | Message-ID의 해시값 (파일명용) |

##### 예외 / 오류
| 조건 | 오류 코드 | 설명 |
|------|-----------|------|
| 본문 추출 실패 | ERR_MAIL_PARSE | 메일 파싱 오류 |

### 처리 흐름

1. **메일 제목 추출**: rawMail.subject 그대로 사용
2. **본문 추출**:
   - rawMail.isHtml === true이면 HTML-to-Text 변환 (MAIL-R-012)
   - HTML 태그 제거, 본문 텍스트만 추출
3. **첨부파일 무시**: 본문 텍스트만 처리 (MAIL-R-013)
4. **Message-ID 해시 생성**: SHA-256 해시의 앞 8자리 (파일명 충돌 방지)
5. **결과 조합**: subject + textBody를 하나의 텍스트로 구성

### 출력 텍스트 형식

```
Subject: {메일 제목}
Date: {수신 일시}
---
{본문 텍스트}
```

### 구현 가이드

- **패턴**: Pure Function - lib/mail/mail-parser.ts
- **HTML-to-Text**: html-to-text 라이브러리 또는 간단한 태그 스트리핑 구현
  - `<br>`, `<p>` -> 줄바꿈, `<a href="">` -> URL 추출, 나머지 태그 제거
- **인코딩**: UTF-8 통일
- **성능**: 단일 메일 단위 처리, 메모리 이슈 없음
- **외부 의존성**: HTML 파서 라이브러리 (선택적), crypto (해시)

### 관련 기능
- **이 기능을 호출하는 기능**: TERM-BATCH-001
- **이 기능이 호출하는 기능**: CMN-FS-001 (결과 텍스트 임시 저장은 DATA-FILE-001이 담당)

### 테스트 시나리오

| 시나리오 | 입력 조건 | 기대 결과 |
|----------|-----------|-----------|
| 텍스트 메일 | isHtml=false, body="안녕하세요" | 본문 그대로 추출 |
| HTML 메일 | isHtml=true, body="<p>안녕</p>" | "안녕" 텍스트 추출 |
| 첨부파일 포함 | 본문 + 첨부 | 본문만 추출, 첨부 무시 |
| 빈 본문 | body="" | 빈 문자열 반환 |
| 복잡한 HTML | 테이블, 이미지 포함 HTML | 텍스트만 추출 |
