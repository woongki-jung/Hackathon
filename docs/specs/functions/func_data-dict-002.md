# 용어 사전 백업 기능 정의

## 개요

- **기능 목적**: 앱 종료 시 용어 사전 데이터를 백업하고, 오래된 백업 파일을 정리한다.
- **적용 범위**: 앱 종료 이벤트에서 호출된다.

---

## DATA-DICT-002: 용어 사전 백업

### 기본 정보

| 항목 | 내용 |
|------|------|
| 기능명 | 용어 사전 백업 |
| 분류 | 도메인 특화 로직 |
| 레이어 | Infrastructure |
| 트리거 | 앱 종료 시 |
| 관련 정책 | POL-DATA (DATA-05) |

### 입력 / 출력

#### 입력 (Input)

| 파라미터 | 타입 | 필수 | 설명 |
|----------|------|------|------|
| dictionaryDir | string | ✅ | 사전 데이터 디렉터리 경로 |

#### 출력 (Output)

| 항목 | 타입 | 설명 |
|------|------|------|
| backupPath | string | 생성된 백업 파일 경로 |
| cleanedCount | int | 삭제된 오래된 백업 파일 수 |

#### 예외 / 오류

| 조건 | 오류 코드 | 설명 |
|------|-----------|------|
| 백업 생성 실패 | ERR_DICT_BACKUP_FAILED | 백업 파일 쓰기 실패 |

### 처리 흐름

1. **백업 디렉터리 보장**: `{dictionaryDir}/backup/` 디렉터리를 확인/생성한다.
2. **백업 파일 생성**: 사전 데이터를 `dictionary_{yyyyMMdd_HHmmss}.bak`으로 복사한다 (DATA-05).
3. **오래된 백업 정리**: CMN-FS-001.CleanupOldFiles로 최근 7개만 유지하고 나머지를 삭제한다 (DATA-05).
4. **결과 반환**: 백업 경로와 정리된 파일 수를 반환한다.

### 구현 가이드

- **패턴**: CMN-FS-001의 CleanupOldFiles 활용.
- **동시성**: 앱 종료 시 단일 스레드에서 실행. Graceful shutdown 보장 필요.

### 관련 기능

- **이 기능을 호출하는 기능**: 앱 종료 이벤트 핸들러
- **이 기능이 호출하는 기능**: CMN-FS-001, CMN-LOG-001

### 테스트 시나리오

| 시나리오 | 입력 조건 | 기대 결과 |
|----------|-----------|-----------|
| 정상 백업 | 유효한 사전 데이터 | 백업 파일 생성 |
| 파일명 형식 | 2026-03-13 15:00:00 | "dictionary_20260313_150000.bak" |
| 오래된 백업 정리 | 10개 기존 백업 | 3개 삭제, 7개 유지 |
| 첫 백업 | 백업 디렉터리 미존재 | 디렉터리 생성 후 백업 |
