# 파일 시스템 관리 기능 정의

## 개요

- **기능 목적**: 디렉터리 자동 생성, 파일 이동, 디스크 용량 확인 등 파일 시스템 관련 공통 처리를 제공한다.
- **적용 범위**: 분석 요청 파일 생성(DATA-FILE-001), 처리 완료 파일 관리(DATA-FILE-002), 용어 사전 백업(DATA-DICT-002) 등 파일 I/O가 필요한 모든 기능.

---

## CMN-FS-001: 파일 시스템 관리

### 기본 정보

| 항목 | 내용 |
|------|------|
| 기능명 | 파일 시스템 관리 |
| 분류 | 공통 기능 |
| 레이어 | Infrastructure |
| 트리거 | 파일 저장/이동/삭제 시, 앱 시작 시 디렉터리 초기화 |
| 관련 정책 | POL-DATA (DATA-01, DATA-06), POL-DATA 제약사항 |

### 입력 / 출력

#### 디렉터리 보장 (EnsureDirectory)

| 파라미터 | 타입 | 필수 | 설명 |
|----------|------|------|------|
| path | string | ✅ | 보장할 디렉터리 경로 |

출력: 성공 여부 (boolean)

#### 파일 이동 (MoveFile)

| 파라미터 | 타입 | 필수 | 설명 |
|----------|------|------|------|
| sourcePath | string | ✅ | 원본 파일 경로 |
| destinationDir | string | ✅ | 이동 대상 디렉터리 |

출력: 이동된 파일의 전체 경로 (string)

#### 디스크 용량 확인 (CheckDiskSpace)

| 파라미터 | 타입 | 필수 | 설명 |
|----------|------|------|------|
| path | string | ✅ | 확인할 경로 |
| minimumBytes | long | ✅ | 최소 필요 용량(바이트) |

출력: 충분 여부 (boolean), 사용 가능 용량 (long)

#### 오래된 파일 정리 (CleanupOldFiles)

| 파라미터 | 타입 | 필수 | 설명 |
|----------|------|------|------|
| directory | string | ✅ | 대상 디렉터리 |
| pattern | string | ✅ | 파일 패턴 (예: "*.bak") |
| keepCount | int | ✅ | 유지할 최신 파일 수 |

출력: 삭제된 파일 수 (int)

#### 예외 / 오류

| 조건 | 오류 코드 | 설명 |
|------|-----------|------|
| 권한 없음 | ERR_FS_ACCESS_DENIED | 디렉터리/파일 접근 권한 없음 |
| 경로 무효 | ERR_FS_INVALID_PATH | 잘못된 경로 형식 |
| 디스크 부족 | ERR_FS_DISK_LOW | 디스크 용량 최소 기준 미달 |
| 파일 미존재 | ERR_FS_FILE_NOT_FOUND | 이동 대상 파일 없음 |

### 처리 흐름

**디렉터리 보장**:
1. 경로 유효성 검사 (한글/공백 포함 경로 지원, DATA-01).
2. 디렉터리 존재 여부 확인.
3. 미존재 시 재귀적으로 생성.
4. 생성 실패 시 ERR_FS_ACCESS_DENIED 반환.

**디스크 용량 확인**:
1. 지정 경로의 드라이브 정보를 조회한다.
2. 사용 가능 용량이 minimumBytes 미만이면 false를 반환한다.
3. 1GB 미만 시 CMN-NOTI-001을 통해 트레이 알림을 발행한다 (POL-DATA 제약사항).

### 구현 가이드

- **패턴**: `IFileSystemService` 인터페이스로 추상화하여 테스트에서 가상 파일 시스템으로 대체 가능.
- **보안**: 경로 순회(Path Traversal) 공격 방지를 위해 기본 디렉터리 외부 접근을 차단한다.
- **동시성**: 파일 이동 시 파일 잠금 상태 확인. 재시도 없이 실패를 보고한다.

### 관련 기능

- **이 기능을 호출하는 기능**: DATA-FILE-001, DATA-FILE-002, DATA-DICT-002
- **이 기능이 호출하는 기능**: CMN-LOG-001, CMN-NOTI-001

### 테스트 시나리오

| 시나리오 | 입력 조건 | 기대 결과 |
|----------|-----------|-----------|
| 디렉터리 생성 | 미존재 경로 | 디렉터리 생성, true 반환 |
| 이미 존재하는 디렉터리 | 기존 경로 | true 반환 (무작업) |
| 한글 경로 지원 | "C:\분석\요청" | 정상 생성 |
| 파일 이동 성공 | 유효한 소스, 유효한 대상 | 이동 완료, 신규 경로 반환 |
| 파일 이동 실패 | 미존재 소스 | ERR_FS_FILE_NOT_FOUND |
| 디스크 용량 충분 | 10GB 사용 가능, 최소 1GB | true |
| 디스크 용량 부족 | 500MB 사용 가능, 최소 1GB | false, 트레이 알림 발행 |
| 오래된 파일 정리 | 10개 bak, keepCount=7 | 3개 삭제 |
