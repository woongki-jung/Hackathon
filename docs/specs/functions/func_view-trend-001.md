# 빈도 트렌드 조회 기능 정의

## 개요

- **기능 목적**: 최근 7일간 발견 빈도가 증가한 상위 용어를 집계하여 반환한다.
- **적용 범위**: 용어사전 뷰어 메인 화면의 "최근 빈도 높은 용어" 섹션.

---

## VIEW-TREND-001: 빈도 트렌드 조회

### 기본 정보

| 항목 | 내용 |
|------|------|
| 기능명 | 빈도 트렌드 조회 |
| 분류 | 도메인 특화 로직 |
| 레이어 | Domain |
| 트리거 | 메인 화면 로드 시 |
| 관련 정책 | POL-UI (UI-04) |

### 입력 / 출력

#### 입력 (Input)

| 파라미터 | 타입 | 필수 | 설명 | 유효성 규칙 |
|----------|------|------|------|-------------|
| topCount | int | | 상위 용어 수 | 기본값 10 (UI-04) |
| daysRange | int | | 집계 기간(일) | 기본값 7 (UI-04) |

#### 출력 (Output)

| 항목 | 타입 | 설명 |
|------|------|------|
| trendingTerms | TrendEntry[] | 빈도 상위 용어 목록 |

**TrendEntry 구조**

| 필드 | 타입 | 설명 |
|------|------|------|
| entry | DictionaryEntry | 용어 사전 항목 |
| recentCount | int | 최근 daysRange일간 발견 횟수 증가량 |

### 처리 흐름

1. **기준 시점 계산**: 현재 시각에서 daysRange일 전 시점을 계산한다.
2. **대상 필터링**: `lastSeenAt`이 기준 시점 이후인 용어를 필터링한다.
3. **빈도 기준 정렬**: sourceCount 기준 내림차순으로 정렬한다 (UI-04).
4. **상한 적용**: topCount개만 반환한다.

### 구현 가이드

- **패턴**: 순수 함수. DATA-DICT-001에서 전체 목록을 조회한 후 필터링/정렬.
- **성능**: 사전 규모가 커지면 인덱스 기반 쿼리로 전환 고려.

### 관련 기능

- **이 기능을 호출하는 기능**: UI 메인 화면
- **이 기능이 호출하는 기능**: DATA-DICT-001

### 테스트 시나리오

| 시나리오 | 입력 조건 | 기대 결과 |
|----------|-----------|-----------|
| 정상 조회 | 최근 7일간 활동 용어 15개 | 상위 10개 반환 |
| 활동 없음 | 7일간 갱신된 용어 없음 | 빈 목록 |
| 동일 빈도 | 동일 sourceCount | 안정적인 정렬 (lastSeenAt 최신 우선) |
| topCount 조정 | topCount=5 | 5개만 반환 |
