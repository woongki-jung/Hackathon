# 트레이 알림 발송 기능 정의

## 개요

- **기능 목적**: 시스템 트레이 풍선 알림(Balloon Notification)을 통해 사용자에게 이벤트를 알린다.
- **적용 범위**: 메일 수신 오류(POL-MAIL MAIL-06), 분석 완료, 디스크 용량 부족 등 사용자 알림이 필요한 모든 상황.

---

## CMN-NOTI-001: 트레이 알림 발송

### 기본 정보

| 항목 | 내용 |
|------|------|
| 기능명 | 트레이 알림 발송 |
| 분류 | 공통 기능 |
| 레이어 | Infrastructure |
| 트리거 | 알림이 필요한 이벤트 발생 시 |
| 관련 정책 | POL-UI (UI-02) |

### 입력 / 출력

#### 입력 (Input)

| 파라미터 | 타입 | 필수 | 설명 | 유효성 규칙 |
|----------|------|------|------|-------------|
| notificationType | enum | ✅ | 알림 유형 (AnalysisComplete / MailError / DiskLow) | |
| title | string | ✅ | 알림 제목 | 최대 64자 |
| message | string | ✅ | 알림 본문 | 최대 256자 |
| durationSeconds | int | | 표시 시간(초) | 기본값 5 |

#### 출력 (Output)

| 항목 | 타입 | 설명 |
|------|------|------|
| displayed | boolean | 알림 표시 성공 여부 |

#### 예외 / 오류

| 조건 | 오류 코드 | 설명 |
|------|-----------|------|
| 트레이 아이콘 미초기화 | ERR_NOTI_TRAY_NOT_READY | 트레이 아이콘이 아직 생성되지 않음 |

### 처리 흐름

1. **트레이 아이콘 상태 확인**: 트레이 아이콘이 초기화되었는지 확인한다.
2. **메시지 구성**: notificationType에 따라 사전 정의된 메시지 템플릿을 적용한다.
   - AnalysisComplete: "{n}건의 새로운 용어가 추가되었습니다" (UI-02)
   - MailError: "메일 서버 연결에 문제가 있습니다" (UI-02)
   - DiskLow: "저장 공간이 부족합니다" (UI-02)
3. **풍선 알림 표시**: 5초간 표시한다 (UI-02).
4. **클릭 이벤트 등록**: 알림 클릭 시 메인 윈도우를 표시하는 이벤트를 연결한다 (UI-02).

### 구현 가이드

- **패턴**: 메시지 템플릿을 Dictionary로 관리. `INotificationService` 인터페이스 제공.
- **동시성**: UI 스레드에서 실행해야 하므로 Dispatcher를 통한 마샬링이 필요하다.
- **외부 의존성**: Windows 트레이 아이콘 API (NotifyIcon 또는 WinUI AppNotification)

### 관련 기능

- **이 기능을 호출하는 기능**: TERM-BATCH-001, CMN-FS-001, MAIL-RECV-001
- **이 기능이 호출하는 기능**: 없음

### 테스트 시나리오

| 시나리오 | 입력 조건 | 기대 결과 |
|----------|-----------|-----------|
| 분석 완료 알림 | AnalysisComplete, n=5 | "5건의 새로운 용어가 추가되었습니다" 표시 |
| 메일 오류 알림 | MailError | "메일 서버 연결에 문제가 있습니다" 표시 |
| 디스크 부족 알림 | DiskLow | "저장 공간이 부족합니다" 표시 |
| 트레이 미초기화 | 아이콘 생성 전 호출 | ERR_NOTI_TRAY_NOT_READY |
