# 앱 상태 정보 조회 기능 정의

## 개요

- **기능 목적**: 총 등록 용어 수, 마지막 메일 확인 시각 등 앱 전반의 상태 정보를 조회한다.
- **적용 범위**: 용어사전 뷰어 메인 화면 하단의 상태 표시줄.

---

## VIEW-STATE-001: 앱 상태 정보 조회

### 기본 정보

| 항목 | 내용 |
|------|------|
| 기능명 | 앱 상태 정보 조회 |
| 분류 | 도메인 특화 로직 |
| 레이어 | Application |
| 트리거 | 메인 화면 로드 시, 주기적 갱신 |
| 관련 정책 | POL-UI (UI-04) |

### 입력 / 출력

#### 입력 (Input)

없음

#### 출력 (Output)

| 항목 | 타입 | 설명 |
|------|------|------|
| totalTermCount | int | 총 등록 용어 수 |
| lastMailCheckAt | DateTime? | 마지막 메일 확인 시각 |
| statusText | string | 표시용 상태 문자열 |

### 처리 흐름

1. **용어 수 조회**: DATA-DICT-001에서 총 등록 용어 수를 가져온다.
2. **마지막 확인 시각 조회**: 메일 폴링 서비스의 마지막 실행 시각을 가져온다.
3. **상태 문자열 구성**: "총 {n}개 용어 등록됨 | 마지막 메일 확인: {시각}" 형식으로 구성한다 (UI-04).
4. **결과 반환**: 상태 정보를 반환한다.

### 구현 가이드

- **패턴**: 각 데이터 소스에서 조회하여 조합하는 간단한 Application Service.
- **성능**: 빈번하게 호출될 수 있으므로 캐싱(30초 정도) 고려.

### 관련 기능

- **이 기능을 호출하는 기능**: UI 상태 표시줄
- **이 기능이 호출하는 기능**: DATA-DICT-001, CMN-CFG-001

### 테스트 시나리오

| 시나리오 | 입력 조건 | 기대 결과 |
|----------|-----------|-----------|
| 정상 조회 | 500개 용어, 최근 확인 10분 전 | "총 500개 용어 등록됨 | 마지막 메일 확인: 14:20" |
| 초기 상태 | 용어 0건, 확인 이력 없음 | "총 0개 용어 등록됨 | 마지막 메일 확인: -" |
