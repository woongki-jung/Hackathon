# 용어사전 뷰어 메인 화면 정의

## 화면 기본 정보

| 항목 | 내용 |
|------|------|
| 화면 코드 | MAIN-001 |
| 화면명 | 용어사전 뷰어 메인 |
| 경로/위치 | 메인 윈도우 (기본 상태) |
| 접근 권한 | 로컬 사용자 |
| 관련 정책 | POL-UI (UI-04, UI-05, UI-07), POL-TERM |
| 이전 화면 | TRAY-001 트레이 아이콘 (좌클릭), TRAY-002 메뉴 "열기", MAIN-003 용어 상세 (뒤로) |
| 다음 화면 | MAIN-002 검색 결과 목록 (검색어 입력), MAIN-003 용어 상세 (빈도 용어 클릭) |

---

## 화면 목적

사용자가 용어사전에 빠르게 접근할 수 있는 시작점을 제공한다. 검색 기능을 통해 원하는 용어를 찾고, 최근 빈도가 높은 용어를 바로가기로 확인할 수 있다. 앱의 현재 상태(총 용어 수, 마지막 메일 확인 시각)를 확인할 수 있다.

## 화면 구성 요소

### 타이틀 바 영역

| 컴포넌트 | 타입 | 표시 조건 | 설명 |
|----------|------|-----------|------|
| 윈도우 제목 | TitleBar | 항상 | 앱 이름 표시 (예: "용어해설 도구") |
| 최소화 버튼 | Button | 항상 | 윈도우 최소화 |
| 최대화 버튼 | Button | 항상 | 윈도우 최대화/복원 |
| 닫기(X) 버튼 | Button | 항상 | 윈도우 숨김 (트레이로 최소화, 앱 종료 아님) |

### 검색 영역 (화면 중앙 상단)

| 컴포넌트 | 타입 | 표시 조건 | 설명 |
|----------|------|-----------|------|
| 검색 입력창 | TextBox | 항상 | 용어 검색을 위한 입력 필드, 중앙 배치, 플레이스홀더: "용어를 검색하세요" |
| 검색 아이콘 | Icon | 항상 | 검색창 내부 또는 우측 돋보기 아이콘 |

### 빈도 높은 용어 영역 (검색창 아래)

| 컴포넌트 | 타입 | 표시 조건 | 설명 |
|----------|------|-----------|------|
| 섹션 제목 | Label | 용어 데이터가 있을 때 | "최근 자주 발견된 용어" |
| 용어 태그 목록 | ChipGroup | 용어 데이터가 있을 때 | 최근 7일간 source_count 증가량 상위 10개 용어를 태그/칩 형태로 표시 |
| 용어 태그 | Chip | 각 용어별 | 용어 원문 표시, 카테고리별 색상 구분 |
| 빈 상태 메시지 | Label | 용어 데이터가 없을 때 | "아직 등록된 용어가 없습니다. 메일 수신 후 자동으로 용어가 추가됩니다." |

### 상태 바 영역 (화면 하단)

| 컴포넌트 | 타입 | 표시 조건 | 설명 |
|----------|------|-----------|------|
| 총 용어 수 | Label | 항상 | "총 {n}개 용어 등록됨" |
| 구분자 | Label | 항상 | "|" |
| 마지막 확인 시각 | Label | 메일 확인 이력 있을 때 | "마지막 메일 확인: {yyyy-MM-dd HH:mm}" |
| 미확인 안내 | Label | 메일 확인 이력 없을 때 | "마지막 메일 확인: -" |

## 세부 기능 정의

### 용어 검색

| 항목 | 내용 |
|------|------|
| 트리거 | 사용자가 검색 입력창에 텍스트를 입력 |
| 처리 | 300ms 디바운스 적용 후 용어 사전에서 검색 실행 |
| 성공 결과 | 화면이 MAIN-002 검색 결과 목록 상태로 전환 |
| 실패 결과 | 해당 없음 (로컬 검색) |

**검색 규칙 (POL-UI UI-05):**
- 검색 대상: 용어 원문(term) 및 해설(description)
- 검색 방식: 부분 일치 (입력 문자열이 포함된 모든 항목)
- 대소문자 구분 없음
- 정렬 순서: 용어 원문 정확 일치 > 용어 부분 일치 > 해설 포함
- 디바운스: 300ms

### 빈도 높은 용어 클릭

| 항목 | 내용 |
|------|------|
| 트리거 | 사용자가 빈도 높은 용어 태그(칩)를 클릭 |
| 처리 | 해당 용어의 상세 화면(MAIN-003) 표시 |
| 성공 결과 | 용어 상세 화면으로 전환 |
| 실패 결과 | 해당 없음 |

### 윈도우 닫기 (X 버튼)

| 항목 | 내용 |
|------|------|
| 트리거 | 사용자가 타이틀 바 X 버튼을 클릭 |
| 처리 | 메인 윈도우를 숨기고 트레이 영역으로 최소화 |
| 성공 결과 | 윈도우가 숨겨지고 트레이 아이콘만 유지 |
| 실패 결과 | 해당 없음 |

## 화면 상태 정의

| 상태 | 발생 조건 | 표시 내용 |
|------|-----------|-----------|
| 초기 로딩 | 윈도우 최초 표시 | 검색창 표시, 빈도 용어 로딩 중 |
| 데이터 있음 (정상) | 용어 사전에 데이터가 존재 | 검색창 + 빈도 높은 용어 태그 목록 + 상태 바 |
| 데이터 없음 | 용어 사전이 비어있음 | 검색창 + "아직 등록된 용어가 없습니다" 안내 + 상태 바 (총 0개) |
| 검색 입력 중 | 사용자가 검색어를 입력 | MAIN-002 검색 결과 화면으로 전환 |

## 화면 전환 정의

| 트리거 | 이동 화면 | 전환 방식 | 전달 데이터 |
|--------|-----------|-----------|-------------|
| 검색 입력창에 텍스트 입력 | MAIN-002 검색 결과 목록 | 화면 내 뷰 전환 | 검색어 문자열 |
| 빈도 높은 용어 태그 클릭 | MAIN-003 용어 상세 | 화면 내 뷰 전환 | 용어 ID/term |
| X 버튼 클릭 | TRAY-001 트레이 아이콘 (유지) | 윈도우 숨김 | 없음 |

|------|-------------|--------|
| 해당 없음 | - | 메인 화면 자체에서는 별도 알림 없음, 알림은 TRAY-001 풍선 알림으로 제공 |

## 비즈니스 규칙 및 표시 조건

- 검색창은 구글 검색창과 유사하게 화면 중앙 상단에 배치한다 (POL-UI UI-04).
- 빈도 높은 용어는 최근 7일간 source_count 증가량 기준 상위 10개를 표시한다 (POL-UI UI-04).
- 카테고리별로 태그/칩의 색상을 구분한다: EMR / Business / Abbreviation (POL-UI UI-06).
- 윈도우 크기: 기본 800x600, 최소 640x480 (POL-UI UI-07).
- 사용자가 조정한 윈도우 크기와 위치는 다음 실행 시 복원한다 (POL-UI UI-07).
- 시스템 다크/라이트 모드를 따른다 (POL-UI 제약 사항).
- 관련 정책 문서: docs/specs/policies/policy_pol-ui.md, docs/specs/policies/policy_pol-term.md
