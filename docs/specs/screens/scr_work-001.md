## 화면 기본 정보

| 항목 | 내용 |
|------|------|
| 화면 코드 | WORK-001 |
| 화면명 | 업무지원 상세 |
| 라우트 | `/work/:id` |
| App Router 파일 | `app/(authenticated)/work/[id]/page.tsx` |

## 화면 목적

사용자가 개별 메일의 분석 결과를 상세하게 확인할 수 있다. 메일의 핵심 요약, 후속 작업 제안, 추출된 용어 목록을 포함하여 업무 이해도를 높이는 데 도움을 준다.

## 세부 기능 정의

### 분석 결과 상세 조회

| 항목 | 내용 |
|------|------|
| 트리거 | 화면 진입 시 자동 실행 (URL의 `:id` 파라미터 사용) |
| 처리 | 분석 대기열에서 해당 ID의 상세 결과를 조회 |
| 관련 API | ANAL-002 `GET /api/analysis/history`에서 개별 항목 (또는 id 기반 조회) |
| 관련 기능 | TERM-BATCH-001 |
| 성공 결과 | 메일 정보, 요약, 후속 작업, 추출 용어 표시 |
| 실패 결과 | 오류 메시지 표시 |

**처리 흐름:**
1. URL에서 `:id` 파라미터를 추출한다.
2. `GET /api/analysis/history` 또는 개별 조회 API로 해당 분석 결과를 조회한다.
3. 로딩 중 스켈레톤 UI를 표시한다.
4. 성공 시 각 영역에 데이터를 배치한다.
5. `status`가 `completed`가 아닌 경우 요약/후속 작업 영역을 숨기고 상태에 맞는 안내를 표시한다.
6. 존재하지 않는 ID인 경우 404 상태를 표시한다.

### 용어 태그 클릭

| 항목 | 내용 |
|------|------|
| 트리거 | 사용자가 추출된 용어 태그를 클릭 |
| 처리 | 해당 용어의 상세 페이지로 이동 |
| 관련 API | 없음 (클라이언트 네비게이션) |
| 성공 결과 | DICT-002 용어 상세 화면으로 이동 |
| 실패 결과 | 없음 |

## 화면 상태 정의

| 상태 | 발생 조건 | 표시 내용 |
|------|-----------|-----------|
| 초기 로딩 | 화면 진입 시 | 스켈레톤 UI (메일 정보 + 요약 + 작업 목록 영역) |
| 정상 (분석 완료) | status=completed | 모든 영역 데이터 표시 |
| 분석 중 | status=processing | 메일 정보 + "분석이 진행 중입니다" 안내 + 스피너 |
| 대기 중 | status=pending | 메일 정보 + "분석 대기 중입니다" 안내 |
| 분석 실패 | status=failed | 메일 정보 + 오류 메시지 + 재시도 횟수 |
| 데이터 없음 | 존재하지 않는 ID | "요청하신 분석 결과를 찾을 수 없습니다." + 대시보드 이동 링크 |
| 오류 | API 호출 실패 | 오류 메시지 + 재시도 버튼 |

## 비즈니스 규칙 및 표시 조건

- 분석 결과의 `status`에 따라 표시되는 영역이 다르다: `completed`일 때만 요약/후속 작업/용어 태그를 표시한다.
- `actionItems`는 JSON 배열로 저장되어 있으며, 최대 5개 항목을 표시한다 (TERM-R-011).
- `summary`는 최대 500자까지이다 (TERM-R-010).
- 날짜/시간은 "YYYY-MM-DD HH:mm" 형식으로 표시한다 (UI-R-015).
- 용어 태그는 카테고리에 따라 시각적으로 구분한다 (emr/business/abbreviation/general).
- 관련 정책 문서: `docs/specs/policies/policy_pol-ui.md`, `docs/specs/policies/policy_pol-term.md`
