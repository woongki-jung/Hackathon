# 트레이 컨텍스트 메뉴 화면 정의

## 화면 기본 정보

| 항목 | 내용 |
|------|------|
| 화면 코드 | TRAY-002 |
| 화면명 | 트레이 컨텍스트 메뉴 |
| 경로/위치 | 트레이 아이콘 우클릭 시 표시되는 팝업 메뉴 |
| 접근 권한 | 로컬 사용자 |
| 관련 정책 | POL-UI (UI-01) |
| 이전 화면 | TRAY-001 트레이 아이콘 (우클릭) |
| 다음 화면 | MAIN-001 메인 화면, SET-001 환경설정, 앱 종료 |

---

## 화면 목적

트레이 아이콘에서 앱의 주요 기능에 빠르게 접근할 수 있는 메뉴를 제공한다. 메인 윈도우 열기, 환경설정, 메일 즉시 확인, 앱 종료 기능을 제공한다.

## 세부 기능 정의

### "열기" 클릭 - 메인 윈도우 표시

| 항목 | 내용 |
|------|------|
| 트리거 | 사용자가 "열기" 메뉴 항목을 클릭 |
| 처리 | 메인 윈도우(MAIN-001)를 표시하거나 포커스 |
| 성공 결과 | 메인 윈도우가 전면에 표시되고 메뉴가 닫힘 |
| 실패 결과 | 해당 없음 |

### "환경설정" 클릭 - 환경설정 다이얼로그 열기

| 항목 | 내용 |
|------|------|
| 트리거 | 사용자가 "환경설정" 메뉴 항목을 클릭 |
| 처리 | 환경설정 다이얼로그(SET-001)를 모달로 표시 |
| 성공 결과 | 환경설정 다이얼로그가 표시되고 메뉴가 닫힘 |
| 실패 결과 | 해당 없음 |

### "메일 즉시 확인" 클릭 - 즉시 폴링

| 항목 | 내용 |
|------|------|
| 트리거 | 사용자가 "메일 즉시 확인" 메뉴 항목을 클릭 |
| 처리 | 메일함 폴링을 즉시 1회 실행 (예정된 주기와 무관하게) |
| 성공 결과 | 폴링 실행 후 신규 메일이 있으면 분석 처리, 완료 시 풍선 알림 표시 |
| 실패 결과 | 메일 서버 연결 실패 시 풍선 알림으로 오류 안내 |

**처리 흐름:**
1. 사용자가 "메일 즉시 확인" 클릭
2. 메뉴 닫힘
3. 메일함 폴링 즉시 실행 (POL-MAIL 정책 적용)
4. 신규 메일 발견 시 분석 요청 파일 저장 (POL-DATA 적용)
5. 용어 분석 배치 실행 (POL-TERM 적용)
6. 완료 시 풍선 알림 표시 (TRAY-001)

### "종료" 클릭 - 앱 종료

| 항목 | 내용 |
|------|------|
| 트리거 | 사용자가 "종료" 메뉴 항목을 클릭 |
| 처리 | 데이터 백업 수행 후 앱 완전 종료 |
| 성공 결과 | 트레이 아이콘 제거, 프로세스 종료 |
| 실패 결과 | 백업 실패 시 경고 표시 후 종료 여부 재확인 |

**처리 흐름:**
1. 사용자가 "종료" 클릭
2. 용어 사전 데이터 백업 실행 (POL-DATA DATA-05)
3. 백업 성공 시 트레이 아이콘 제거 및 앱 종료
4. 백업 실패 시 경고 메시지 표시, 사용자 확인 후 종료

## 알림/피드백 정의

| 상황 | 피드백 유형 | 메시지 |
|------|-------------|--------|
| 메일 즉시 확인 완료 (신규 용어) | 풍선 알림 (정보) | "{n}건의 새로운 용어가 추가되었습니다" |
| 메일 즉시 확인 완료 (신규 없음) | 풍선 알림 (정보) | "새로운 메일이 없습니다" |
| 메일 즉시 확인 실패 | 풍선 알림 (경고) | "메일 서버 연결에 문제가 있습니다" |
| 종료 시 백업 실패 | 확인 다이얼로그 | "데이터 백업에 실패했습니다. 그래도 종료하시겠습니까?" |

## 비즈니스 규칙 및 표시 조건

- 메뉴 항목 순서는 "열기", "환경설정", "메일 즉시 확인", 구분선, "종료" 순으로 고정한다 (POL-UI UI-01).
- 앱 종료는 이 메뉴의 "종료" 항목을 통해서만 가능하다 (POL-UI UI-01).
- 메일 즉시 확인 실행 중에는 해당 메뉴 항목을 비활성화하여 중복 실행을 방지한다.
- 관련 정책 문서: docs/specs/policies/policy_pol-ui.md
