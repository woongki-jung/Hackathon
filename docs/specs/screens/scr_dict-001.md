## 화면 목적

사용자가 백엔드에서 생성된 용어 해설 데이터를 검색하고 조회할 수 있다. 구글 검색창과 유사한 단일 검색 입력 폼을 제공하고, 최근 발견 빈도가 높은 용어를 바로가기 형태로 표시한다.

## 세부 기능 정의

### 용어 검색

| 항목 | 내용 |
|------|------|
| 트리거 | 사용자가 검색 입력에 키워드를 입력 (300ms 디바운스, UI-R-012) |
| 처리 | FTS5 전문 검색으로 용어명 및 해설에서 검색 |
| 관련 API | DICT-001 `GET /api/dictionary/search?q={keyword}&category={category}&page=1&limit=20` |
| 관련 기능 | VIEW-SEARCH-001, DATA-DICT-001 |
| 성공 결과 | 검색 결과 목록 표시 |
| 실패 결과 | 오류 메시지 표시 |

**처리 흐름:**
1. 사용자가 검색 입력에 키워드를 입력한다.
2. 300ms 디바운스를 적용하여 입력이 멈추면 검색 API를 호출한다 (UI-R-012).
3. 검색어가 비어 있으면 검색 결과 영역을 숨기고 트렌드 영역을 표시한다.
4. 로딩 중 스켈레톤 UI를 표시한다.
5. 검색 결과를 목록으로 표시한다. 0건이면 "검색 결과가 없습니다." 메시지를 표시한다 (UI-R-013).
6. 카테고리 필터 변경 시 동일 키워드로 재검색한다.
7. 페이지네이션으로 다음 페이지를 조회할 수 있다.

### 카테고리 필터 변경

| 항목 | 내용 |
|------|------|
| 트리거 | 사용자가 카테고리 필터를 변경 |
| 처리 | 현재 검색 키워드 + 선택된 카테고리로 재검색 |
| 관련 API | DICT-001 (category 파라미터 추가) |
| 성공 결과 | 필터링된 검색 결과 표시, 페이지 1로 리셋 |
| 실패 결과 | 오류 메시지 표시 |

### 빈도 트렌드 조회

| 항목 | 내용 |
|------|------|
| 트리거 | 화면 진입 시 자동 실행 |
| 처리 | 빈도 상위 10개 용어 조회 |
| 관련 API | DICT-002 `GET /api/dictionary/trending?limit=10` |
| 관련 기능 | VIEW-TREND-001 |
| 성공 결과 | 트렌드 용어 태그 표시 |
| 실패 결과 | 트렌드 영역에 오류 메시지 표시 |

| 항목 | 내용 |
|------|------|
| 트리거 | 사용자가 검색 결과 항목을 클릭 |

## 화면 상태 정의

| 상태 | 발생 조건 | 표시 내용 |
|------|-----------|-----------|
| 초기 | 화면 진입 (검색어 없음) | 검색 입력(비어 있음) + 트렌드 용어 영역 |
| 검색 중 | 디바운스 후 API 호출 중 | 검색 결과 영역 스켈레톤 UI |
| 검색 결과 있음 | 검색 결과 1건 이상 | 검색 결과 목록 + 페이지네이션 |
| 검색 결과 없음 | 검색 결과 0건 | "검색 결과가 없습니다." (UI-R-013) |
| 트렌드 로딩 | 트렌드 데이터 조회 중 | 태그 스켈레톤 UI |
| 트렌드 비어 있음 | 등록된 용어 0건 | "등록된 용어가 없습니다." |
| 오류 | API 호출 실패 | 오류 메시지 + 재시도 버튼 |

## 비즈니스 규칙 및 표시 조건

- 검색은 300ms 디바운스를 적용하여 실시간 검색 결과를 표시한다 (UI-R-012).
- 검색 결과 목록은 한 페이지당 최대 20건이며, 초과 시 페이지네이션을 제공한다 (UI-R-014).
- 검색 정렬 기준: FTS5 관련도 우선, 동일 관련도이면 빈도 내림차순.
- 빈도 트렌드는 빈도 내림차순으로 정렬한 상위 10개를 표시한다 (UI-R-017).
- 검색어가 비어 있으면 검색 결과 영역을 숨기고 트렌드 영역을 기본으로 표시한다.
- 검색 결과의 해설은 200자까지만 표시하고 말줄임 처리한다.
- 관련 정책 문서: `docs/specs/policies/policy_pol-ui.md`, `docs/specs/policies/policy_pol-term.md`
