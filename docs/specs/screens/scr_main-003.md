# 용어 상세 화면 정의

## 화면 기본 정보

| 항목 | 내용 |
|------|------|
| 화면 코드 | MAIN-003 |
| 화면명 | 용어 상세 |
| 경로/위치 | 메인 윈도우 (상세 상태) |
| 접근 권한 | 로컬 사용자 |
| 관련 정책 | POL-UI (UI-06), POL-TERM (TERM-03, TERM-04) |
| 이전 화면 | MAIN-002 검색 결과 목록 (용어 선택), MAIN-001 메인 화면 (빈도 용어 클릭), MAIN-003 자기 자신 (관련 용어 클릭) |
| 다음 화면 | MAIN-002 검색 결과 목록 ("목록으로" 클릭), MAIN-003 자기 자신 (관련 용어 클릭) |

---

## 화면 목적

사용자가 선택한 용어의 상세 해설 정보를 제공한다. 카테고리, 요약, 상세 설명, 관련 용어, 발견 이력 등 용어에 대한 모든 정보를 한 화면에서 확인할 수 있다.

---

## 화면 구성 요소

### 타이틀 바 영역

MAIN-001과 동일 (앱 공통 타이틀 바).

### 네비게이션 영역 (상단)

| 컴포넌트 | 타입 | 표시 조건 | 설명 |
|----------|------|-----------|------|
| "목록으로" 버튼 | Button | 검색 결과에서 진입한 경우 | 검색 결과 목록(MAIN-002)으로 복귀 |
| "홈으로" 버튼 | Button | 메인 화면에서 직접 진입한 경우 | 메인 화면(MAIN-001)으로 복귀 |

### 용어 헤더 영역

| 컴포넌트 | 타입 | 표시 조건 | 설명 |
|----------|------|-----------|------|
| 용어 원문 | Label (큰 텍스트) | 항상 | 용어 원문 표시 (제목 수준) |
| 카테고리 태그 | Badge | 항상 | EMR / Business / Abbreviation, 카테고리별 색상 |

### 해설 영역

| 컴포넌트 | 타입 | 표시 조건 | 설명 |
|----------|------|-----------|------|
| 1줄 요약 | Label (강조) | 해설이 있을 때 | 50자 이내 요약 텍스트 |
| 상세 설명 | TextBlock | 해설이 있을 때 | 200자 이내 상세 해설 텍스트 |
| 해설 미완료 안내 | Label | 해설이 없을 때 (해설 미완료 상태) | "해설을 준비 중입니다. 다음 분석 시 자동 생성됩니다." |

### 관련 용어 영역

| 컴포넌트 | 타입 | 표시 조건 | 설명 |
|----------|------|-----------|------|
| 섹션 제목 | Label | 관련 용어가 있을 때 | "관련 용어" |
| 관련 용어 태그 목록 | ChipGroup | 관련 용어가 있을 때 | 관련 용어를 태그/칩 형태로 표시 |
| 관련 용어 태그 | Chip (클릭 가능) | 각 관련 용어별 | 클릭 시 해당 용어 상세로 이동 |

### 발견 이력 영역

| 컴포넌트 | 타입 | 표시 조건 | 설명 |
|----------|------|-----------|------|
| 섹션 제목 | Label | 항상 | "발견 이력" |
| 발견 횟수 | Label | 항상 | "총 {source_count}회 발견" |
| 최초 발견 일시 | Label | 항상 | "최초 발견: {first_seen_at}" |
| 최근 발견 일시 | Label | 항상 | "최근 발견: {last_seen_at}" |

### 상태 바 영역 (화면 하단)

MAIN-001과 동일 (앱 공통 상태 바).

## 세부 기능 정의

### 용어 상세 조회

| 항목 | 내용 |
|------|------|
| 트리거 | 화면 진입 시 (MAIN-002에서 용어 선택 또는 MAIN-001에서 빈도 용어 클릭) |
| 처리 | 용어 사전에서 해당 용어의 전체 데이터 조회 |
| 성공 결과 | 용어 상세 정보 표시 |
| 실패 결과 | 해당 없음 (로컬 데이터 조회) |

**표시 데이터 (POL-UI UI-06):**
- 용어 원문 (term)
- 카테고리 (category): EMR / Business / Abbreviation
- 1줄 요약 (description 중 요약 부분)
- 상세 설명 (description 중 상세 부분)
- 관련 용어 목록 (클릭 시 해당 용어로 이동)
- 발견 횟수 (source_count)
- 최초 발견 일시 (first_seen_at)
- 최근 발견 일시 (last_seen_at)

### 관련 용어 이동

| 항목 | 내용 |
|------|------|
| 트리거 | 사용자가 관련 용어 태그를 클릭 |
| 처리 | 클릭한 용어의 상세 화면으로 전환 (MAIN-003 자기 자신 갱신) |
| 성공 결과 | 새 용어의 상세 정보 표시 |
| 실패 결과 | 해당 용어가 사전에 없는 경우 "해당 용어 정보를 찾을 수 없습니다" 안내 |

### 목록으로 복귀

| 항목 | 내용 |
|------|------|
| 트리거 | 사용자가 "목록으로" 버튼을 클릭 |
| 처리 | 이전 검색 결과 목록(MAIN-002)으로 복귀, 검색어 및 스크롤 위치 유지 |
| 성공 결과 | 검색 결과 목록 화면 표시 |
| 실패 결과 | 해당 없음 |

## 화면 상태 정의

| 상태 | 발생 조건 | 표시 내용 |
|------|-----------|-----------|
| 정상 (해설 있음) | 용어에 해설이 존재 | 모든 정보 표시 |
| 해설 미완료 | 용어가 등록되었으나 해설이 아직 생성되지 않음 | 용어명 + 카테고리 + "해설을 준비 중입니다" 안내 + 발견 이력 |
| 용어 없음 | 관련 용어 클릭 시 해당 용어가 사전에 없음 | "해당 용어 정보를 찾을 수 없습니다" 안내 |

---

## 화면 전환 정의

| 트리거 | 이동 화면 | 전환 방식 | 전달 데이터 |
|--------|-----------|-----------|-------------|
| "목록으로" 버튼 클릭 | MAIN-002 검색 결과 목록 | 화면 내 뷰 전환 (뒤로) | 이전 검색어 |
| "홈으로" 버튼 클릭 | MAIN-001 메인 화면 | 화면 내 뷰 전환 (뒤로) | 없음 |
| 관련 용어 태그 클릭 | MAIN-003 용어 상세 (자기 자신) | 화면 내 데이터 갱신 | 용어 ID/term |
| X 버튼 (타이틀 바) 클릭 | TRAY-001 트레이 아이콘 | 윈도우 숨김 | 없음 |

## 알림/피드백 정의

| 상황 | 피드백 유형 | 메시지 |
|------|-------------|--------|
| 해설 미완료 상태 | 인라인 안내 | "해설을 준비 중입니다. 다음 분석 시 자동 생성됩니다." |
| 관련 용어 없음 (사전 미등록) | 인라인 안내 | "해당 용어 정보를 찾을 수 없습니다" |

---

## 비즈니스 규칙 및 표시 조건

- 카테고리별 색상을 구분하여 표시한다 (POL-UI UI-06).
- 관련 용어 클릭 시 해당 용어가 사전에 등록된 경우에만 이동한다 (POL-UI UI-06).
- "목록으로" 버튼으로 복귀 시 이전 검색어와 검색 결과를 유지한다 (POL-UI UI-06).
- 해설 미완료 상태의 용어는 다음 배치 분석 시 자동으로 해설이 생성된다 (POL-TERM TERM-06).
- 관련 정책 문서: docs/specs/policies/policy_pol-ui.md, docs/specs/policies/policy_pol-term.md
