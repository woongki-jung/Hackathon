## 화면 목적

사용자가 서비스의 전반적인 상태를 한눈에 파악하고, 최신 메일 분석 결과를 즉시 확인할 수 있다. 백엔드 서비스의 마지막 실행 시점과 상태를 표시하며, 최신 분석 결과를 상단에 노출하고 이전 이력은 목록으로 제공한다.

공통 레이아웃의 GNB를 표시한다 (spec-screens.md 참조).

| 스케줄러 상태 뱃지 | Badge | 항상 | 실행 중(녹색) / 중지(회색) |

| API 키 설정 상태 | Badge | 항상 | 설정 완료(녹색) / 미설정(빨간색) |
| 마지막 메일 수신 | Label | 이력 있을 때 | "YYYY-MM-DD HH:mm / 성공(3건)" 형식 |

| 수동 메일 확인 버튼 | Button | admin만 | "메일 확인 실행" |

### 최신 분석 결과 영역

| 컴포넌트 | 타입 | 표시 조건 | 설명 |
|----------|------|-----------|------|
| 영역 제목 | Label | 항상 | "최신 분석 결과" |
| 메일 제목 | Label (heading) | 분석 결과 있을 때 | 분석된 메일의 제목 표시 |
| 메일 수신일 | Label (secondary) | 분석 결과 있을 때 | "YYYY-MM-DD HH:mm 수신" (UI-R-015) |
| 요약 텍스트 | Paragraph | 분석 결과 있을 때 | 메일 핵심 내용 요약 (최대 500자) |
| 후속 작업 목록 | Ordered List | 분석 결과 있을 때 | 최대 5개 항목 표시 |
| 추출 용어 수 | Badge | 분석 결과 있을 때 | "8개 용어 추출" 형식 |
| 상세 보기 링크 | Link | 분석 결과 있을 때 | WORK-001로 이동 |
| 빈 상태 | Empty State | 분석 결과 없을 때 | "아직 분석된 메일이 없습니다." + 안내 문구 |

### 분석 이력 목록 영역

| 컴포넌트 | 타입 | 표시 조건 | 설명 |
|----------|------|-----------|------|
| 영역 제목 | Label | 항상 | "이전 분석 이력" |
| 이력 목록 | List | 이력 있을 때 | 최신순 정렬 (UI-R-016), 페이지당 20건 |
| 목록 항목 | ListItem | 이력 있을 때 | 메일 제목, 수신일, 상태 뱃지, 요약 미리보기 |
| 상태 뱃지 | Badge | 이력 있을 때 | completed(녹색)/failed(빨간색)/processing(노란색)/pending(회색) |

| 빈 상태 | Empty State | 이력 없을 때 | "분석 이력이 없습니다." |
| 로딩 상태 | Skeleton | 데이터 로딩 중 | 목록 스켈레톤 UI (UI-R-018) |

### 수동 메일 확인 실행 (관리자 전용)

| 항목 | 내용 |
|------|------|
| 트리거 | 관리자가 "메일 확인 실행" 버튼을 클릭 |
| 처리 | 메일 수신/분석 프로세스를 수동으로 트리거 |
| 관련 API | MAIL-002 `POST /api/mail/check` |
| 관련 기능 | SCHED-001, TERM-BATCH-001 |
| 성공 결과 | 토스트: "메일 확인이 시작되었습니다." + 상태 갱신 |
| 실패 결과 | 토스트: 오류 메시지 |

**처리 흐름:**
1. 관리자가 "메일 확인 실행" 버튼을 클릭한다.
2. 확인 다이얼로그: "메일 확인을 즉시 실행하시겠습니까?"
3. 확인 시 `POST /api/mail/check` API 호출한다.
4. 이미 실행 중이면 "이미 메일 확인 작업이 진행 중입니다." 토스트를 표시한다.
5. 트리거 성공 시 "메일 확인이 시작되었습니다. 완료까지 시간이 소요될 수 있습니다." 토스트를 표시한다.
6. 5초 후 서비스 상태를 자동 재조회한다.

## 화면 상태 정의

| 상태 | 발생 조건 | 표시 내용 |
|------|-----------|-----------|
| 초기 로딩 | 화면 최초 진입 | 서비스 상태 카드 스켈레톤 + 분석 결과 스켈레톤 + 이력 스켈레톤 |
| 설정 미완료 | IMAP 또는 API 키 미설정 | 경고 배너 표시 + 분석 결과 없음 상태 |
| 분석 결과 없음 | 완료된 분석 없음 | 최신 분석 영역에 빈 상태 일러스트 + 안내 |
| 정상 | 모든 데이터 있음 | 상태 카드 + 최신 분석 + 이력 목록 표시 |
| 오류 | API 호출 실패 | 각 영역별 오류 메시지 + 재시도 버튼 |

## 비즈니스 규칙 및 표시 조건

- 서비스 상태, 최신 분석 결과, 이력 목록은 화면 진입 시 동시에(병렬) 조회한다.
- "수동 메일 확인 실행" 버튼은 관리자(`admin`)에게만 표시한다.
- 설정 미완료 배너의 "설정 페이지로 이동" 링크는 관리자에게만 표시한다. 일반 사용자에게는 "관리자에게 문의하세요." 문구를 표시한다.
- 날짜/시간은 "YYYY-MM-DD HH:mm" 형식으로 표시한다 (UI-R-015).
- 이력 목록은 최신순(내림차순) 정렬한다 (UI-R-016).
- 관련 정책 문서: `docs/specs/policies/policy_pol-ui.md`, `docs/specs/policies/policy_pol-auth.md`
