# 사용자 관리 화면 정의

## 화면 기본 정보

| 항목 | 내용 |
|------|------|
| 화면 코드 | ADMIN-001 |
| 화면명 | 사용자 관리 |
| 라우트 | `/admin/users` |
| App Router 파일 | `app/(authenticated)/admin/users/page.tsx` |
| 접근 권한 | 관리자 전용 (admin). 일반 사용자 접근 시 403 또는 대시보드로 리다이렉트 |
| 관련 정책 | POL-AUTH (AUTH-R-002 ~ AUTH-R-007, AUTH-R-013, AUTH-R-015), POL-DATA (DATA-R-015) |
| 이전 화면 | DASH-001 대시보드 (GNB에서 진입) |
| 다음 화면 | DASH-001 대시보드 (GNB) |

---

## 화면 목적

관리자가 서비스 사용자 계정을 관리할 수 있다. 등록된 사용자 목록을 조회하고, 새 사용자를 등록하거나, 기존 사용자를 삭제(소프트 삭제)할 수 있다.

---

## 세부 기능 정의

### 사용자 목록 조회

| 항목 | 내용 |
|------|------|
| 트리거 | 화면 진입 시 자동 실행 |
| 처리 | 등록된 사용자 전체 목록 조회 (소프트 삭제 제외) |
| 관련 API | USER-001 `GET /api/users` |
| 관련 기능 | CMN-AUTH-001 |
| 성공 결과 | 사용자 목록 테이블에 데이터 표시 |
| 실패 결과 | 오류 메시지 + 재시도 버튼 |

### 사용자 등록

| 항목 | 내용 |
|------|------|
| 트리거 | 사용자가 등록 폼에서 "등록" 버튼을 클릭 |
| 처리 | 새 사용자 계정을 생성 |
| 관련 API | USER-002 `POST /api/users` |
| 관련 기능 | CMN-AUTH-001 |
| 성공 결과 | 토스트: "사용자가 등록되었습니다." + 목록 갱신 + 폼 닫기 |
| 실패 결과 | 폼 내 오류 메시지 표시 |

**입력 유효성 검사 (클라이언트)**:

| 필드 | 유효성 규칙 | 오류 메시지 |
|------|-------------|-------------|
| 아이디 | 필수, 4~20자, 영문소문자/숫자/_ (AUTH-R-003) | "아이디는 4자 이상 20자 이하의 영문 소문자, 숫자, 언더스코어만 허용합니다." |
| 비밀번호 | 필수, 8자 이상, 영문자+숫자+특수문자 각 1자 이상 (AUTH-R-005, AUTH-R-006) | "비밀번호는 8자 이상이며 영문자, 숫자, 특수문자를 각각 1자 이상 포함해야 합니다." |
| 비밀번호 확인 | 비밀번호와 동일 | "비밀번호가 일치하지 않습니다." |
| 역할 | "admin" 또는 "user" | - |

**처리 흐름:**
1. 사용자가 "사용자 등록" 버튼을 클릭하여 등록 폼을 연다.
2. 아이디, 비밀번호, 비밀번호 확인, 역할을 입력한다.
3. "등록" 버튼을 클릭한다.
4. 클라이언트에서 유효성을 검사한다.
5. 유효성 통과 시 `POST /api/users` API를 호출한다.
6. 호출 중 등록 버튼을 로딩 상태로 전환한다.
7. 성공 시 토스트 메시지를 표시하고 목록을 갱신하며 폼을 닫는다.
8. 아이디 중복(409) 시 "이미 존재하는 아이디입니다." 오류를 표시한다.
9. 기타 오류 시 해당 필드에 서버 반환 오류 메시지를 표시한다.

### 사용자 삭제

| 항목 | 내용 |
|------|------|
| 트리거 | 사용자가 목록의 삭제 버튼을 클릭 |
| 처리 | 대상 사용자 계정을 소프트 삭제 |
| 관련 API | USER-003 `DELETE /api/users/:id` |
| 관련 기능 | CMN-AUTH-001 |
| 성공 결과 | 토스트: "사용자가 삭제되었습니다." + 목록에서 해당 사용자 제거 |
| 실패 결과 | 토스트: 오류 메시지 |

**처리 흐름:**
1. 사용자가 삭제 버튼을 클릭한다.
2. 확인 다이얼로그: "'{username}' 사용자를 삭제하시겠습니까? 삭제된 계정은 복구할 수 없습니다."
3. 확인 시 `DELETE /api/users/:id` API를 호출한다.
4. 호출 중 해당 행을 비활성화 처리한다.
5. 성공 시 토스트 메시지를 표시하고 목록을 갱신한다.
6. 실패 시 오류 토스트를 표시한다.

## 비즈니스 규칙 및 표시 조건

- 사용자 관리는 관리자만 가능하다 (AUTH-R-002, AUTH-R-015). 일반 사용자는 이 페이지에 접근할 수 없다.
- 관리자만 새 사용자 계정을 등록할 수 있다 (AUTH-R-002).
- 아이디는 4~20자, 영문소문자/숫자/언더스코어만 허용한다 (AUTH-R-003).
- 아이디는 시스템 내 고유해야 하며, 소프트 삭제된 아이디도 재사용 불가하다 (AUTH-R-004).
- 비밀번호는 8자 이상, 영문자+숫자+특수문자 각 1자 이상이다 (AUTH-R-005, AUTH-R-006).
- 소프트 삭제된 사용자는 목록에 표시하지 않는다.
- 자기 자신의 계정은 삭제할 수 없다 (로그인 중인 관리자 보호).
- 소규모 사용자 수를 가정하여 페이지네이션 없이 전체 목록을 표시한다.
- `passwordHash` 필드는 API 응답에 절대 포함되지 않는다.
- 관련 정책 문서: `docs/specs/policies/policy_pol-auth.md`, `docs/specs/policies/policy_pol-data.md`
