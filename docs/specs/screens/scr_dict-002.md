## 화면 목적

사용자가 특정 용어의 상세 해설 전문과 해당 용어가 추출된 출처 메일 목록을 확인할 수 있다. 용어의 분류, 발견 빈도, 최초/최근 추출 일시 등 메타 정보를 함께 제공한다.

---

## 화면 구성 요소

### GNB 영역
공통 레이아웃의 GNB를 표시한다 (spec-screens.md 참조).

### 헤더 영역

| 컴포넌트 | 타입 | 표시 조건 | 설명 |
|----------|------|-----------|------|
| 뒤로가기 버튼 | Button (icon) | 항상 | 이전 화면으로 이동 |
| 페이지 제목 | Label | 항상 | "용어 상세" |

### 용어 기본 정보 영역

| 컴포넌트 | 타입 | 표시 조건 | 설명 |
|----------|------|-----------|------|
| 용어명 | Label (heading, h1) | 데이터 있을 때 | 용어 이름 (큰 글씨) |
| 카테고리 뱃지 | Badge | category 있을 때 | emr/business/abbreviation/general |
| 발견 빈도 | Label | 데이터 있을 때 | "N회 발견" |
| 최초 추출일 | Label (secondary) | 데이터 있을 때 | "최초 추출: YYYY-MM-DD HH:mm" |
| 최근 갱신일 | Label (secondary) | 데이터 있을 때 | "최근 갱신: YYYY-MM-DD HH:mm" |

### 해설 영역

| 컴포넌트 | 타입 | 표시 조건 | 설명 |
|----------|------|-----------|------|
| 영역 제목 | Label | 데이터 있을 때 | "해설" |
| 해설 본문 | Paragraph (rich text) | 데이터 있을 때 | 용어 해설 전문 (잘라내지 않음) |

### 출처 메일 목록 영역

| 컴포넌트 | 타입 | 표시 조건 | 설명 |
|----------|------|-----------|------|
| 영역 제목 | Label | 데이터 있을 때 | "출처 메일 (N건)" |
| 출처 목록 | List | sources 있을 때 | 메일 제목 + 수신 일시 목록, 최신순 정렬, 최대 10건 |
| 목록 항목 | ListItem | sources 있을 때 | 메일 제목, 수신 일시 표시 |
| 빈 상태 | Label | sources 없을 때 | "출처 메일 정보가 없습니다." |

## 세부 기능 정의

### 용어 상세 조회

| 항목 | 내용 |
|------|------|
| 트리거 | 화면 진입 시 자동 실행 (URL의 `:id` 파라미터 사용) |
| 처리 | 용어 상세 정보 및 출처 메일 목록 조회 |
| 관련 API | DICT-003 `GET /api/dictionary/terms/:id` |
| 관련 기능 | DATA-DICT-001, VIEW-SEARCH-001 |
| 성공 결과 | 용어 기본 정보 + 해설 + 출처 메일 표시 |
| 실패 결과 | 오류 메시지 표시 |

**처리 흐름:**
1. URL에서 `:id` 파라미터를 추출한다.
2. `GET /api/dictionary/terms/:id` API를 호출한다.
3. 로딩 중 스켈레톤 UI를 표시한다.
4. 성공 시 용어 기본 정보, 해설 전문, 출처 메일 목록을 표시한다.
5. 존재하지 않는 ID인 경우 404 상태를 표시한다.

## 화면 상태 정의

| 상태 | 발생 조건 | 표시 내용 |
|------|-----------|-----------|
| 초기 로딩 | 화면 진입 시 | 스켈레톤 UI (용어 정보 + 해설 + 출처 영역) |
| 정상 | 데이터 조회 성공 | 모든 영역 데이터 표시 |
| 데이터 없음 | 존재하지 않는 ID (404) | "요청하신 용어를 찾을 수 없습니다." + 용어사전 이동 링크 |
| 오류 | API 호출 실패 | 오류 메시지 + 재시도 버튼 |

## 비즈니스 규칙 및 표시 조건

- 용어 해설은 전문을 표시한다 (DICT-001 검색 결과의 200자 미리보기와 다름).
- 출처 메일 목록은 `mail_received_at` 기준 최신순으로 최대 10건을 표시한다.
- 날짜/시간은 "YYYY-MM-DD HH:mm" 형식으로 표시한다 (UI-R-015).
- 용어 데이터는 영구 보존되므로 삭제된 용어는 없다 (POL-DATA DATA-R-017).
- 관련 정책 문서: `docs/specs/policies/policy_pol-ui.md`, `docs/specs/policies/policy_pol-term.md`
