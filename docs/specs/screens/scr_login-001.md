## 화면 목적

사용자가 아이디와 비밀번호를 입력하여 서비스에 로그인할 수 있다. 관리자 또는 관리자가 등록한 사용자만 인증이 가능하며, 로그인 성공 시 iron-session 기반 암호화 쿠키가 발급된다.

---

## 화면 구성 요소

### 전체 레이아웃
- GNB를 표시하지 않는다 (POL-UI 예외 사항).
- 로그인 폼을 화면 중앙에 배치한다 (POL-UI UI-R-007).
- 서비스 로고/이름을 폼 상단에 표시한다.

### 로그인 폼 영역

| 컴포넌트 | 타입 | 표시 조건 | 설명 |
|----------|------|-----------|------|
| 서비스 로고/이름 | Label | 항상 | 서비스명 표시 |
| 아이디 입력 | Input (text) | 항상 | placeholder: "아이디 입력", autofocus |
| 비밀번호 입력 | Input (password) | 항상 | placeholder: "비밀번호 입력" |
| 로그인 버튼 | Button (primary) | 항상 | "로그인" 텍스트, Enter 키로도 제출 가능 |
| 오류 메시지 | Alert (error) | 로그인 실패 시 | 통합 오류 메시지 표시 |
| 로딩 스피너 | Spinner | API 호출 중 | 로그인 버튼 내부에 표시, 버튼 비활성화 |

## 세부 기능 정의

### 로그인

| 항목 | 내용 |
|------|------|
| 트리거 | 사용자가 로그인 버튼을 클릭하거나 Enter 키를 누름 |
| 처리 | 아이디/비밀번호로 서버 인증 후 세션 쿠키 발급 |
| 관련 API | AUTH-001 `POST /api/auth/login` |
| 관련 기능 | CMN-AUTH-001, CMN-SESSION-001 |
| 성공 결과 | 대시보드(`/`)로 페이지 이동 |
| 실패 결과 | 폼 상단에 오류 메시지 표시, 비밀번호 필드 초기화 |

**입력 유효성 검사 (클라이언트)**:

| 필드 | 유효성 규칙 | 오류 메시지 |
|------|-------------|-------------|
| 아이디 | 필수, 4~20자, 영문소문자/숫자/_ | "아이디는 4자 이상 20자 이하의 영문 소문자, 숫자, 언더스코어만 허용합니다." |
| 비밀번호 | 필수, 8자 이상 | "비밀번호는 8자 이상이어야 합니다." |

**처리 흐름:**
1. 사용자가 아이디/비밀번호를 입력하고 로그인 버튼을 클릭한다.
2. 클라이언트에서 필수 입력 및 형식 유효성을 검사한다.
3. 유효성 통과 시 `POST /api/auth/login` API를 호출한다.
4. 호출 중 로그인 버튼을 로딩 상태로 전환하고 입력 비활성화한다.
5. 성공 시 `router.push('/')`로 대시보드 이동한다.
6. 실패 시 오류 메시지를 표시하고 비밀번호 필드를 초기화한다.

### 이미 로그인 상태 확인

| 항목 | 내용 |
|------|------|
| 트리거 | 로그인 페이지 진입 시 |
| 처리 | `GET /api/auth/me`로 현재 세션 확인 |
| 관련 API | AUTH-003 `GET /api/auth/me` |
| 성공 결과 | 이미 로그인 상태이면 대시보드(`/`)로 리다이렉트 |
| 실패 결과 | 로그인 폼을 표시 |

## 화면 상태 정의

| 상태 | 발생 조건 | 표시 내용 |
|------|-----------|-----------|
| 초기 | 페이지 로드 완료 | 로그인 폼 표시, 아이디 입력에 autofocus |
| 로딩 | 로그인 API 호출 중 | 버튼 로딩 스피너, 입력 비활성화 |
| 오류 | 로그인 실패 (401) | "아이디 또는 비밀번호가 일치하지 않습니다" (AUTH-R-011) |
| 유효성 오류 | 클라이언트 유효성 검사 실패 | 각 필드 하단에 인라인 오류 메시지 |
| 네트워크 오류 | API 호출 실패 | "일시적인 오류가 발생했습니다. 다시 시도해주세요." |

## 화면 전환 정의

| 트리거 | 이동 화면 | 전환 방식 | 전달 데이터 |
|--------|-----------|-----------|-------------|
| 로그인 성공 | DASH-001 대시보드 | 페이지 이동 (`router.push('/')`) | 없음 (세션 쿠키 자동 설정) |
| 이미 로그인 상태 | DASH-001 대시보드 | 리다이렉트 (`router.replace('/')`) | 없음 |

| 상황 | 피드백 유형 | 메시지 |
|------|-------------|--------|
| 로그인 실패 | 인라인 Alert (error) | "아이디 또는 비밀번호가 일치하지 않습니다." |
| 유효성 오류 | 인라인 필드 오류 | 각 필드별 오류 메시지 |

| 세션 만료 리다이렉트 | URL 파라미터 기반 Alert (info) | "세션이 만료되었습니다. 다시 로그인해주세요." |

## 비즈니스 규칙 및 표시 조건

- 로그인 실패 시 아이디/비밀번호 중 어느 쪽이 틀렸는지 구분하지 않는 통합 메시지를 표시한다 (AUTH-R-011).
- 소프트 삭제된 사용자 또는 비활성 사용자는 로그인이 불가하다.
- 키보드만으로 전체 로그인 흐름을 완료할 수 있어야 한다 (POL-UI UI-R-022).
- 모든 입력 필드에 `<label>` 요소를 연결한다 (POL-UI UI-R-021).
- 세션 만료로 리다이렉트된 경우, URL 쿼리 파라미터(`?expired=true`)를 통해 안내 메시지를 표시한다.
