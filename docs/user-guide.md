# 사용자 가이드

## 서비스 소개

**도메인 사전**은 업무 메일을 자동으로 수신하고 AI(Gemini)로 분석하여 EMR, 비즈니스, 약어 등의 도메인 용어를 추출·해설하는 웹 서비스입니다.

---

## 초기 설정 (관리자)

### 1. 로그인

- 주소: `http://서버주소:3000/login`
- 초기 관리자 계정은 `.env.local`의 `ADMIN_USERNAME` / `ADMIN_PASSWORD`로 설정

### 2. 메일 서버 설정

1. GNB → **환경설정** 클릭
2. **메일 서버 (IMAP)** 섹션에서 입력:
   - IMAP 호스트 (예: `imap.gmail.com`)
   - 포트 (SSL: `993`, 비SSL: `143`)
   - 아이디 (이메일 주소)
   - SSL/TLS 사용 여부
   - 메일 확인 주기 (분, 최소 1분)
3. **연결 테스트** 버튼으로 IMAP 접속 확인
4. 비밀번호·API 키는 서버의 `.env.local` 파일에서 직접 설정

> IMAP 비밀번호는 보안상 화면에서 입력받지 않습니다. 서버의 `.env.local`에 `MAIL_PASSWORD`로 설정하세요.

### 3. Gemini API 설정

1. [Google AI Studio](https://aistudio.google.com/)에서 API 키 발급
2. 서버의 `.env.local`에 `GEMINI_API_KEY` 설정
3. 환경설정 → **AI 분석** 섹션에서 모델명 확인 (기본: `gemini-1.5-pro`)

## 주요 기능 사용법

### 대시보드

- **서비스 상태 카드**: 스케줄러 실행 여부, 메일 서버/API 키 설정 상태, 마지막 실행 시각 확인
- **최신 분석 결과**: 가장 최근에 처리된 메일의 AI 요약과 후속 작업 확인. **상세 보기** 클릭으로 업무지원 상세 이동
- **분석 이력**: 전체 처리 이력 목록. **보기** 클릭으로 상세 확인
- **메일 확인 실행** (관리자 전용): 스케줄러 대기 없이 즉시 메일 수신·분석 트리거

### 업무지원 상세

- 메일 제목, 수신일, 분석 완료일, 상태 배지 확인
- **AI 요약**: 메일 핵심 내용 요약 (최대 500자)
- **후속 작업**: AI가 제안한 액션 아이템 체크리스트. 항목 클릭으로 완료 표시
- **추출된 용어**: 카테고리별 색상 태그. 클릭 시 용어사전 상세로 이동

### 용어사전

- 검색창에 용어 입력 → 300ms 디바운스로 실시간 검색
- **카테고리 필터**: 전체 / EMR / 비즈니스 / 약어 / 일반
- **빈도 트렌드 TOP 10**: 검색어 없을 때 가장 많이 등장한 용어 태그 표시
- 결과 카드 클릭 → 용어 상세(해설 전문, 출처 메일 목록) 확인

### 환경설정 (관리자 전용)

| 항목 | 설명 |
|------|------|
| IMAP 호스트/포트 | 메일 수신 서버 주소 |
| SSL/TLS | 보안 연결 여부 |
| 메일 확인 주기 | 자동 수신 주기 (분) |
| AI 모델 | Gemini 모델명 |
| 연결 테스트 | IMAP 서버 접속 가능 여부 확인 |

### 사용자 관리 (관리자 전용)

- **사용자 등록**: 아이디(4~20자 영문/숫자), 비밀번호(8자 이상, 영문+숫자+특수문자), 권한(관리자/사용자) 설정
- **사용자 삭제**: 소프트 삭제(복구 불가). 자기 자신은 삭제 불가

|----------|------|------|
| EMR | 파란색 | 전자의무기록 관련 용어 |

## 트러블슈팅

### 메일이 수신되지 않을 때

1. 대시보드 → 서비스 상태 카드 → 스케줄러가 "실행 중"인지 확인
2. 환경설정 → 연결 테스트로 IMAP 접속 확인
3. `.env.local`의 `MAIL_PASSWORD`가 올바르게 설정되었는지 확인
4. Gmail 사용 시 앱 비밀번호 사용 여부 확인 (일반 비밀번호 불가)

### 분석이 실패할 때

1. 환경설정 → AI 분석 섹션에서 API 키 설정 여부 확인 (✅ 설정됨)
2. 분석 이력에서 **실패** 항목 클릭 → 오류 메시지 확인
3. 재시도는 최대 3회 자동으로 진행됨

### 용어가 검색되지 않을 때

- FTS5 인덱스 동기화 이슈일 수 있습니다. 관리자가 `POST /api/admin/sync-terms`를 호출하여 파일 동기화 후 재검색
- 완전 일치 단어로 검색해보세요 (FTS5는 접두어 검색 지원)

### 분석은 됐는데 용어사전에 없을 때

- `POST /api/admin/sync-terms` 호출로 파일 재동기화

## 데이터 구조

```
data/
  app.db         ← SQLite DB (전체 데이터)
  mails/         ← 수신 메일 임시 텍스트 (30일 후 자동 삭제)
  terms/         ← 용어 해설집 Markdown 파일
```
